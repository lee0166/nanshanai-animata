# 场景和物品管理筛选问题修复

## 问题描述

用户反馈：

1. 场景管理可以选择筛选剧本，但**场景列表不同步更新**
2. 创建的场景在**左侧区域不显示**
3. 物品管理需要同样检查

## 问题分析

### 角色管理 vs 场景/物品管理的差异

**角色管理**：

- 左侧：AssetList（角色卡片）
- 右侧：CharacterSidebar（角色列表）
- 两侧都有筛选逻辑，同步更新

**场景管理**：

- 左侧：AssetList（场景卡片）
- 右侧：？？？（是否有侧边栏？）

**物品管理**：

- 左侧：AssetList（物品卡片）
- 右侧：ItemSidebar（物品列表）
- 需要检查 ItemSidebar 是否有筛选逻辑

## 需要检查的点

### 1. 场景管理

- [ ] 是否有右侧列表组件？
- [ ] 如果有，是否接收 scriptFilter？
- [ ] AssetList 的筛选逻辑是否正确？

### 2. 物品管理

- [ ] ItemSidebar 是否接收 scriptFilter？
- [ ] ItemSidebar 是否有筛选逻辑？
- [ ] AssetList 的筛选逻辑是否正确？

### 3. 创建后显示问题

- [ ] 创建场景后是否正确刷新列表？
- [ ] 新创建的场景是否有 scriptId？
- [ ] 筛选状态是否影响新创建场景的显示？

## 修复方案

### 方案1：添加场景侧边栏（如果有）

如果场景管理有右侧列表，需要：

1. 添加 scriptFilter 和 scripts props
2. 实现按剧本筛选逻辑

### 方案2：修复 ItemSidebar

如果 ItemSidebar 没有筛选逻辑，需要：

1. 添加 scriptFilter 和 scripts props
2. 实现按剧本筛选逻辑

### 方案3：修复创建后显示

如果创建后左侧不显示，需要：

1. 检查创建后刷新逻辑
2. 检查筛选状态是否匹配新创建的场景

1. 检查 ProjectDetail.tsx 中场景和物品的侧边栏配置
2. 检查 AssetList.tsx 的筛选逻辑
3. 修复发现的问题
4. 测试验证
