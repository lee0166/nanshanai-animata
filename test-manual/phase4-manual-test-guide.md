# Phase 4 迭代优化引擎 - 纯人工手动测试方案

## 测试环境

- **URL**: http://localhost:3000/
- **浏览器**: Chrome/Edge (推荐开启开发者工具)
- **测试内容**: 3000-8000 字短篇小说/剧本

## 测试前准备

### 1. 启动开发服务器

```bash
cd d:\kemeng
npm run dev
```

确认服务运行在 http://localhost:3000/ 或 http://localhost:3001/

### 2. 准备测试小说

建议使用包含以下"问题"的测试内容，以验证优化引擎：

- **角色不一致**: 同一角色在不同段落有不同描述
- **场景问题**: 缺少时间/地点信息
- **描述不完整**: 角色或场景描述过于简单

**示例测试内容结构**:

```
标题: 测试剧本
字数: 约5000字
角色: 4-6个
场景: 8-12个
```

---

## 测试用例 1: 基础功能验证

### 步骤 1.1: 创建新项目

**操作**:

1. 打开 http://localhost:3000/
2. 点击"新建项目"
3. 填写项目名称: "Phase4测试项目"
4. 选择"剧本/小说"类型

**预期结果**:

- 项目创建成功
- 自动跳转到项目详情页
- 左侧显示角色/场景/物品/片段/资源 Tab

**如果失败**:

- 提供浏览器控制台错误截图
- 描述点击后的页面状态

---

### 步骤 1.2: 上传测试小说

**操作**:

1. 在项目详情页点击"上传剧本"
2. 选择准备好的测试小说文件（.txt 或 .docx）
3. 点击"开始解析"

**预期结果**:

- 文件上传成功
- 显示解析进度条
- 进度条显示阶段: metadata → characters → scenes → refinement → shots

**关键观察点**:

- 是否出现 "refinement" 阶段？
- refinement 阶段耗时多久？
- 进度条是否卡在某一阶段？

**如果失败**:

- 提供进度条截图
- 按 F12 打开控制台，查看 Network 和 Console 标签
- 提供控制台错误信息

---

## 测试用例 2: 迭代优化功能验证

### 步骤 2.1: 验证 Refinement 阶段执行

**操作**:

1. 等待解析完成
2. 查看浏览器控制台日志（F12 → Console）

**预期结果**:
控制台应显示以下日志：

```
[ScriptParser] Starting iterative refinement...
[IterativeRefinementEngine] Starting refinement process...
[IterativeRefinementEngine] Initial quality score: XX
[IterativeRefinementEngine] === Iteration 1/X ===
[IterativeRefinementEngine] Found X violations
[IterativeRefinementEngine] Quality score: XX
[IterativeRefinementEngine] Generated X actions
[IterativeRefinementEngine] Applied: X, Skipped: X, Failed: X
[IterativeRefinementEngine] Refinement successful: XX -> XX (+XX)
```

**如果未看到 refinement 日志**:

- 确认代码已集成（检查 scriptParser.ts 中是否有 refinement 相关代码）
- 提供完整的控制台日志截图

---

### 步骤 2.2: 验证优化结果保存

**操作**:

1. 解析完成后，刷新页面
2. 点击"剧本管理" Tab
3. 查看解析状态

**预期结果**:

- 解析状态显示"已完成"
- 剧本内容已正确分段
- 角色/场景/分镜数据已生成

**验证优化是否生效**:

1. 对比优化前后的角色描述
2. 检查场景信息是否补全（时间、地点）

---

## 测试用例 3: 质量报告 UI 验证

### 步骤 3.1: 查看质量报告

**操作**:

1. 解析完成后，在剧本管理页面查找"质量报告"或"优化报告"
2. 点击报告卡片展开

**预期结果**:

- 显示 RefinementReportCard 组件
- 包含三个标签页：概览、迭代详情、完整报告
- 概览页显示：迭代次数、执行时间、初始/最终分数、质量改进进度条

**如果未看到报告**:

- 提供剧本管理页面截图
- 检查控制台是否有组件渲染错误

---

### 步骤 3.2: 验证报告内容

**操作**:

1. 点击"概览"标签
2. 查看统计卡片（迭代次数、执行时间、初始分数、最终分数）
3. 查看质量改进进度条
4. 查看统计信息（发现违规、生成动作、应用修正等）

**预期结果**:

- 迭代次数 ≥ 1
- 初始分数和最终分数有数值
- 质量改进进度条显示百分比
- 统计信息有具体数字

**操作**:

1. 点击"迭代详情"标签
2. 展开某个迭代卡片

**预期结果**:

- 显示每次迭代的详细信息
- 包含：质量分数变化、违规数量、修正动作统计

**操作**:

1. 点击"完整报告"标签

**预期结果**:

- 显示文本格式的完整报告
- 包含执行摘要、统计信息、迭代详情

## 测试用例 4: 一致性检查验证

### 步骤 4.1: 准备有问题的测试内容

创建包含以下问题的测试小说：

```
第一章
张三，一个高大威猛的年轻人，走在街上。
...
（几段后）
张三，身材矮小，躲在角落里。
```

**操作**:

1. 上传包含角色描述矛盾的测试小说
2. 执行解析
3. 查看控制台日志

**预期结果**:
控制台显示：

```
[IterativeRefinementEngine] Found X violations
  [Violation 1] character_inconsistency: 角色描述不一致
```

---

### 步骤 4.2: 验证自动修正

**操作**:

1. 查看解析后的角色信息
2. 检查角色描述是否统一

**预期结果**:

- 角色描述应保持一致（或更详细）
- 不应存在明显矛盾

---

## 问题排查指南

### 情况 1: Refinement 阶段未执行

**检查点**:

1. 控制台是否有 `[ScriptParser] Starting iterative refinement...` 日志？
2. ScriptParser 配置是否启用 refinement？

**需要提供的信息**:

- 完整的控制台日志（从解析开始到结束）
- scriptParser.ts 中 `enableIterativeRefinement` 配置值

---

### 情况 2: Refinement 执行但无改进

**检查点**:

1. 初始质量分数是否已很高（>85）？
2. 是否有违规被发现？
3. 是否生成了修正动作？

**需要提供的信息**:

- 控制台 refinement 相关日志
- 质量报告截图

---

### 情况 3: UI 组件不显示

**检查点**:

1. 组件是否正确导入？
2. 是否有渲染错误？

**需要提供的信息**:

- 页面截图
- 控制台错误信息
- React DevTools 组件树截图

---

### 情况 4: 解析过程卡住

**检查点**:

1. 卡在哪个阶段？
2. 网络请求是否正常？

**需要提供的信息**:

- 进度条截图
- Network 标签页的请求状态
- 控制台错误日志

---
