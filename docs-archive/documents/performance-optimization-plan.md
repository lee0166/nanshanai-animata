## 二、详细优化方案

### 优化 1: 合并全局上下文提取调用 (P0)

**问题**: 当前需要 3次独立的 LLM 调用

```typescript
// 当前实现 (GlobalContextExtractor.ts)
const [storyResult, visualResult, eraResult] = await Promise.allSettled([
  this.extractStoryContext(content), // LLM Call 1
  this.extractVisualContext(content), // LLM Call 2
  this.extractEraContext(content), // LLM Call 3
]);
```

**方案**: 合并为 1次调用，一次性提取所有上下文

```typescript
// 优化后实现
const globalContext = await this.extractAllContext(content); // 1次调用
```

**实现步骤**:

1. 创建新的合并 Prompt，同时请求 story + visual + era
2. 修改 `extract()` 方法，使用单次调用
3. 保持向后兼容（保留原方法作为 fallback）

**预期收益**: 从 110秒 → 15-20秒（减少 85%）

---

### 优化 2: 解除串行执行限制 (P0)

**问题**: 强制串行执行

```typescript
// ScriptParser.ts Line 181-196
const CONFIG = {
  concurrency: 1, // ⚠️ 强制串行
  callDelay: 1000, // ⚠️ 每次调用间隔1秒
  // ...
};
```

**方案**: 增加并发，减少调用延迟

```typescript
// 优化后
const CONFIG = {
  concurrency: 3, // ✅ 3个并发
  callDelay: 0, // ✅ 无延迟（或100ms）
  // ...
};
```

**实现步骤**:

1. 修改 `CONFIG.concurrency` 从 1 到 3
2. 修改 `CONFIG.callDelay` 从 1000 到 0（或100ms）
3. 测试并发稳定性

**预期收益**: 角色/场景提取时间减少 60-70%

---

### 优化 3: 优化 Prompt 长度 (P1)

**问题**: Prompt 过长，增加模型处理时间

**当前 Prompt 长度**:

- `extractStoryContext`: ~8000字符输入
- `extractVisualContext`: ~5000字符输入
- `extractEraContext`: ~5000字符输入

**方案**: 精简 Prompt，减少冗余信息

1. 移除重复的描述
2. 简化 JSON Schema 说明
3. 使用更简洁的示例

**预期收益**: 每次调用减少 10-20% 时间

---

### 优化 4: 添加性能监控 (P1)

**方案**: 在关键路径添加详细的性能日志

```typescript
// 添加性能监控装饰器
@PerformanceMonitor('extractGlobalContext')
async extract(content: string): Promise<GlobalContext> {
  // ...
}
```

**监控点**:

1. 每个 LLM 调用的耗时
2. 每个解析阶段的耗时
3. 缓存命中率
4. 并发执行情况

**预期收益**: 便于后续持续优化和问题诊断

---

### 优化 5: 智能缓存策略 (P2)

**当前状态**: LRU 缓存已启用，但首次解析必然未命中

**方案**: 预计算常见风格的全局上下文

1. 古装/现代/科幻等常见风格预置模板
2. 用户级缓存持久化到 IndexedDB
3. 相似内容检测（文本相似度）

**预期收益**: 缓存命中率提升至 30-50%

## 四、风险评估与回滚方案

### 风险 1: 合并调用后输出质量下降

**缓解措施**:

- 保留原方法作为 fallback
- A/B 测试对比输出质量
- 逐步切换

### 风险 2: 并发增加导致限流

**缓解措施**:

- 可配置的并发数
- 动态调整机制
- 错误重试

### 风险 3: 网络不稳定时并发失败

**缓解措施**:

- 自动降级到串行
- 指数退避重试

### 回滚方案

所有修改都有对应的 git commit，可以随时回滚：

```bash
git reset --hard HEAD~1  # 回滚到上一个检查点
```
