# 剧本管理页面滚动问题修复计划

## 问题描述

用户在剧本管理页面发现：

1. **分镜列表**：当场景数量达到4个及以上时，分镜列表区域显示不全且鼠标滚动无效
2. **角色标签页**：需要检查是否存在同样的问题
3. **场景标签页**：需要检查是否存在同样的问题
4. **道具标签页**：需要检查是否存在同样的问题

## 我的理解确认

根据代码分析，我理解您描述的问题如下：

### 页面结构

```
剧本管理页面 (ScriptManager.tsx)
├── 解析状态卡片
├── Tabs 标签页容器
│   ├── 原文 (source)
│   ├── 角色 (characters) → CharacterMapping 组件
│   ├── 场景 (scenes) → SceneMapping 组件
│   ├── 道具 (items) → ItemMapping 组件
│   └── 分镜 (shots) → ShotList 组件 (包裹在 Card > CardBody 中)
```

### 问题根因分析

#### 1. 分镜列表 (ShotList) 的问题

在 [ScriptManager.tsx L893-914](file:///d:/kemeng/views/ScriptManager.tsx#L893-L914)：

```tsx
<Tab key="shots" title={...}>
  <Card>
    <CardBody className="h-[420px] overflow-y-auto">  {/* 固定高度420px + 滚动 */}
      <ShotList ... />
    </CardBody>
  </Card>
</Tab>
```

**问题**：

- 分镜列表被包裹在 `CardBody` 中，设置了固定高度 `h-[420px]` 和 `overflow-y-auto`
- 这个高度是固定的，当内容超过420px时可以滚动
- **但是** `ShotList` 组件内部使用了 HeroUI 的 `Table` 组件，它有自己的滚动机制
- 当场景数量≥4时，表格内容可能超出了容器，但滚动事件可能被 `Table` 组件拦截或处理不当

#### 2. 角色/场景/道具标签页的问题

查看 [CharacterMapping.tsx L206-355](file:///d:/kemeng/components/ScriptParser/CharacterMapping.tsx#L206-L355)、[SceneMapping.tsx L121-279](file:///d:/kemeng/components/ScriptParser/SceneMapping.tsx#L121-L279)、[ItemMapping.tsx L133-262](file:///d:/kemeng/components/ScriptParser/ItemMapping.tsx#L133-L262)：

```tsx
// CharacterMapping 组件
<div className="space-y-4">
  {/* Header */}
  <div className="flex items-center justify-between">...</div>

  {/* Character Cards Grid */}
  <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
    {scriptCharacters.map((char) => (...))}
  </div>
</div>
```

**问题**：

- 角色、场景、道具组件**没有设置任何高度限制和滚动容器**
- 它们只是简单的 `div` 包裹，依赖父级容器进行滚动
- 在 ScriptManager 中，这些组件直接作为 Tab 的内容，没有额外的滚动容器

对比分镜列表：

```tsx
// 分镜有滚动容器
<CardBody className="h-[420px] overflow-y-auto">
  <ShotList ... />
</CardBody>

// 角色/场景/道具没有滚动容器
<Tab key="characters" ...>
  <CharacterMapping ... />  {/* 直接渲染，无滚动容器 */}
</Tab>
```

### 根本原因

1. **不一致的滚动处理**：分镜列表有固定高度和滚动，但角色/场景/道具没有
2. **Tabs 组件的默认行为**：HeroUI 的 Tabs 组件可能没有正确处理内部内容的溢出
3. **Table 组件的冲突**：ShotList 中的 Table 组件可能与外层容器的滚动产生冲突

## 修复方案

### 方案一：统一所有标签页的滚动处理（推荐）

为所有标签页添加一致的滚动容器，确保内容超出时可以滚动。

**修改文件**：`views/ScriptManager.tsx`

1. **角色标签页**：添加 `h-[calc(100vh-280px)] overflow-y-auto`
2. **场景标签页**：添加 `h-[calc(100vh-280px)] overflow-y-auto`
3. **道具标签页**：添加 `h-[calc(100vh-280px)] overflow-y-auto`
4. **分镜标签页**：调整高度计算方式，与其他标签页一致

### 方案二：修复 ShotList 的 Table 滚动问题

**修改文件**：`components/ScriptParser/ShotList.tsx`

1. 检查 Table 组件的 `isHeaderSticky` 或类似属性
2. 确保 Table 的滚动与外层容器协调
3. 或者将 Table 替换为自定义的可滚动表格

### 方案三：全局 Tabs 容器滚动

**修改文件**：`views/ScriptManager.tsx`

将滚动容器上移到 Tabs 级别，让所有 Tab 内容共享同一个滚动区域。

## 具体修复步骤

### 步骤 1：统一所有标签页的滚动容器

