### 1.2 2026年关键发现与行业趋势

#### 革命性技术突破
1. **双分支扩散Transformer架构**（Seedance 2.0）
   - 首次实现视频和音频在单次生成中同步完成
   - 打破"先画后音"的传统模式
   - 环境音效、角色呼吸、衣物摩擦声等细节自动匹配

2. **音画一体化成为行业标准**
   - 从"先生成画面，后配音"向"音画同出"全面转变
   - 口型、音效、BGM与画面实时对齐，同步率达到99.8%
   - 无需后期合成

3. **四模态输入能力**
   - 支持多达12路同时参考（9张图片、9段视频、3段音频）
   - 一张参考图定风格、一段视频标动作、一段音频定基调、一句文案补细节
   - 通过@提及系统精确控制素材用途

4. **双声道立体声技术**
   - 高仿真、沉浸式的音效生成
   - 支持背景音乐、环境音效或人物解说等音频多轨并行输出
   - 精准对齐画面节奏

5. **成本与效率革命**
   - 单镜头成本从3000元→3元内
   - 短剧单集成本从5万→千元级
   - 制作周期从7天→3小时
   - 效率直接翻倍

#### 对本项目的启示
1. **优先利用现有视频模型的音频能力** - 项目已有Vidu、火山引擎Seedance等视频模型，部分已支持原生音频生成
2. **分步实施** - 先完善独立音频功能（TTS、音效、音乐），再逐步整合到音画一体化
3. **UI优先** - 基于现有预留UI框架快速迭代，保持与主流平台一致的用户体验

#### T1.0 Vidu/Seedance原生音频生成集成【2026新增】

**文件**: `services/video/VideoGenerationService.ts`、`config/models.ts`

**现有状态**: Vidu和Seedance模型已配置`supportsAudioGeneration: true`

**技术要点**（基于Seedance 2.0双分支架构）：
- 视频和音频在单次生成中同步完成
- 支持双声道立体声
- 无需后期合成

**实施内容**:

```typescript
// 在 VideoGenerationService.ts 中
interface VideoGenerationOptions {
  // ... 现有参数
  generateAudio?: boolean;           // ← 新增：是否生成原生音频
  audioStyle?: 'ambient' | 'full';   // ← 新增：音频风格
  audioReferencePaths?: string[];    // ← 新增：音频参考（最多3个）
}

async generateVideo(
  prompt: string,
  options: VideoGenerationOptions,
  modelConfig: ModelConfig
): Promise<GeneratedVideo> {
  // 检查模型是否支持原生音频生成
  const supportsNativeAudio = modelConfig.capabilities?.supportsAudioGeneration;
  
  if (options.generateAudio && !supportsNativeAudio) {
    throw new Error('该模型不支持原生音频生成');
  }

  // 构建请求参数
  const requestParams = {
    prompt,
    // ... 其他参数
    ...(options.generateAudio && {
      generate_audio: true,
      audio_style: options.audioStyle || 'full',
      // 如果是Seedance 2.0，支持音频参考
      ...(options.audioReferencePaths && options.audioReferencePaths.length > 0 && {
        audio_references: options.audioReferencePaths
      })
    })
  };

  // 调用视频生成API
  const response = await this.callVideoAPI(requestParams, modelConfig);
  
  // 返回包含音频的视频
  return {
    ...response,
    hasNativeAudio: options.generateAudio && supportsNativeAudio,
    audioTracks: options.generateAudio ? ['background', 'effects', 'dialogue'] : []
  };
}
```

### 4.2 两种模式的选择策略

```typescript
// 在视频生成时自动选择最佳模式
function getAudioGenerationStrategy(modelConfig: ModelConfig): 'native' | 'separate' {
  // 如果模型支持原生音频生成，优先使用
  if (modelConfig.capabilities?.supportsAudioGeneration) {
    return 'native';
  }
  // 否则使用独立音频生成
  return 'separate';
}
```
