# 优化建议文字含义解释与优化计划

## 用户问题

用户询问优化建议中的以下文字具体指什么：

- "发现一个严重问题，建议先行修复"
- "发现一个警告，建议优化"
- "还有一项"

## 当前代码分析

根据 `services/parsing/QualityAnalyzer.ts` 第717-753行，优化建议的生成逻辑如下：

### 1. [重要] - 严重问题建议

```typescript
if (issue.type === 'error') {
  recommendations.push(`[重要] ${issue.suggestion}`);
}
```

- 来源：error 级别的问题（如缺少分镜、重复角色名等）

### 2. [建议] - 警告优化建议

```typescript
else if (issue.type === 'warning') {
  recommendations.push(`[建议] ${issue.suggestion}`);
}
```

- 来源：warning 级别的问题（如角色描述不完整、场景描述过短等）

### 3. [建议] - 统计信息补充建议

```typescript
// 场景未生成分镜
recommendations.push(`[重要] 还有${statistics.scenesWithoutShots}个场景未生成分镜...`);

// 角色缺少描述
recommendations.push(`[建议] ${statistics.charactersWithoutDescription}个角色缺少详细描述...`);

// 分镜丰富度
recommendations.push(`[建议] 平均每场景分镜数较少(${statistics.avgShotsPerScene}个)...`);
```

## 问题分析

当前建议文字的**问题**：

1. 文字过于简略，用户看不懂具体指什么
2. 缺乏具体上下文信息（哪些角色？哪些场景？）
3. "还有一项" 这样的描述不友好

## 优化方案

### 方案：增强建议文字的描述

修改 `generateRecommendations` 方法，使其生成更具体的建议文字：

```typescript
// 优化前：
`[重要] 还有${statistics.scenesWithoutShots}个场景未生成分镜，建议完成全部分镜生成`
// 优化后：
`[重要] 场景"咖啡厅相遇"、"办公室"缺少分镜，请生成分镜后再试`;
```

### 实施步骤

1. 修改 `QualityAnalyzer.ts` 中的 `generateRecommendations` 方法
2. 让建议包含更具体的上下文信息
3. 添加类型前缀让用户更易理解
   - "🔴 严重" 代替 "[重要]"
   - "🟡 优化" 代替 "[建议]"
   - "💡 提示" 代替空前缀
