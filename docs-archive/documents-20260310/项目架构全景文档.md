## 一、系统概览

### 1.1 项目定位

**Kmeng AI Animata** 是一个**本地优先（Local-First）的 AI 影视资产生成平台**，采用纯前端架构，无需后端服务器。

**核心价值**：

- 🎬 完整的短剧制作工作流（剧本→分镜→关键帧→视频）
- 🏠 完全本地运行，数据隐私最大化
- 🤖 多 AI 提供商支持（火山引擎、Vidu、ModelScope、OpenAI兼容）
- 💡 智能剧本解析与角色/场景提取

## 四、关键设计决策

### 4.1 Local-First 架构

**选择原因**：

- ✅ 数据隐私最大化（不上传任何数据到云端）
- ✅ 无需服务器维护成本
- ✅ 离线可用
- ✅ 符合创作者对素材安全的需求

**实现方式**：

- File System Access API（首选）- 用户选择本地文件夹
- OPFS（Origin Private File System）- 沙盒模式 fallback
- IndexedDB - 持久化文件夹句柄

### 4.2 文本清洗 v3.0 系统性重构

**问题**：之前采用"清除非法字符"的方式，需要穷举所有非法字符组合，无法覆盖所有情况。

**解决方案**：转变为"保留合法字符"的系统性方案

- 定义完整的合法字符集（中文、英文、数字、常用标点、换行）
- 清除所有其他字符
- 规则从14条简化到7条
- 测试用例从43个重构到27个，全部通过

### 4.3 批量创建竞态条件修复

**问题**：Promise.all 并发执行 + 每个创建操作独立更新状态 → 后更新覆盖前更新

**解决方案**：

1. 提取纯创建函数 - 只创建资产，不更新状态
2. 批量创建 - 串行执行 + 统一状态更新（只更新一次）

### 4.4 模型配置统一管理

**设计**：`config/models.ts` 作为单一真实来源（1700+行，40+模型）

- LLM 模型：26个（OpenAI、阿里云、火山引擎、DeepSeek、Kimi、智谱等）
- 图像模型：7个（火山引擎、Vidu、ModelScope）
- 视频模型：9个（Vidu、火山引擎）

**特性**：

- 能力定义（`ModelCapabilities`）
- 参数配置（`ModelParameter`）
- 可见性/隐藏条件
- 提供商特定配置

## 六、开发参考指南

### 6.1 如何添加新功能

1. **更新类型定义**（`types.ts`）- 如果需要新的数据结构
2. **实现服务层**（`services/`）- 核心业务逻辑
3. **创建组件**（`components/`）- UI 展示
4. **添加页面**（`views/`）- 如果需要新页面
5. **更新路由**（`App.tsx`）- 如果需要新路由
6. **添加翻译**（`locales.ts`）- 支持多语言
7. **更新文档**（本文档和全局状态文档）

### 6.2 如何添加新 AI 模型

1. **在 `config/models.ts` 中添加模型配置**
2. **确保 `ModelCapabilities` 正确定义**
3. **配置 `ModelParameter` 参数列表**
4. **如果需要新 Provider，在 `services/ai/providers/` 中创建**
5. **在 `services/ai/ModelRouter.ts` 中添加路由逻辑**（如需要）
6. **更新全局状态文档**
