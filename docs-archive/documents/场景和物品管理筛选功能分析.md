# 场景和物品管理筛选功能分析

## 当前状态检查

### 角色管理 ✅ 已实现

- 剧本筛选器在标题行
- 按剧本筛选角色
- 左右两侧同步

### 场景管理 ❌ 未实现

- 没有剧本筛选功能
- 场景资产没有 `scriptId` 字段

### 物品管理 ❌ 未实现

- 没有剧本筛选功能
- 物品资产没有 `scriptId` 字段

## 问题分析

### 数据结构差异

**角色 (CharacterAsset)**：

```typescript
interface CharacterAsset extends Asset {
  scriptId?: string;  // ✅ 有剧本关联
  ...
}
```

**场景 (SceneAsset)**：

```typescript
interface SceneAsset extends Asset {
  // ❌ 没有 scriptId 字段
  ...
}
```

**物品 (ItemAsset)**：

```typescript
interface ItemAsset extends Asset {
  // ❌ 没有 scriptId 字段
  itemType: ItemType;
  ...
}
```

### 创建流程差异

**角色创建**：

- 从剧本解析页面创建
- 创建时传入 `scriptId`
- 关联到具体剧本

**场景/物品创建**：

- 从资源管理页面直接创建
- 没有传入 `scriptId`
- 不关联剧本

## 实现方案

### 方案1：给场景和物品添加剧本关联（推荐）

**优点**：

- 数据模型统一
- 筛选功能完整
- 符合用户直觉

**实施步骤**：

1. 修改 `types.ts`，给 `SceneAsset` 和 `ItemAsset` 添加 `scriptId` 字段
2. 修改场景/物品创建逻辑，支持选择所属剧本
3. 在场景/物品管理页面添加剧本筛选器
4. 实现按剧本筛选逻辑

**复杂度**：中等

### 方案2：只添加筛选器，不关联剧本

**优点**：

- 实现简单
- 不影响现有数据

**缺点**：

- 筛选结果永远是"全部"
- 功能不完整

**复杂度**：低

## 推荐方案

**方案1：给场景和物品添加剧本关联**

这样能实现：

1. 场景/物品创建时关联剧本
2. 场景/物品管理页面按剧本筛选
3. 与角色管理保持一致的交互体验
