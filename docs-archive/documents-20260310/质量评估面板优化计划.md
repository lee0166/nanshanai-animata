# 质量评估面板优化计划

## 问题诊断

### 当前质量评估面板存在的问题

#### 1. 问题与优化建议的对应关系不清晰

**现状问题**：

- 顶部显示 "4个警告 1个提示" 的统计标签
- 下方"优化建议"列表展示4条建议
- 用户不清楚：这4条建议是否对应4个警告？提示是什么？

**根因分析**：

```typescript
// QualityAnalyzer.ts 第152-158行
const violations: RuleViolation[] = allIssues.map(issue => ({
  ruleId: `quality_${issue.type}`, // 所有问题使用相同的ruleId前缀
  ruleName: '质量检查', // 统一的ruleName
  severity: issue.type,
  message: issue.target ? `[${issue.target}] ${issue.message}` : issue.message,
  suggestion: issue.suggestion,
}));
```

问题：所有质量检查问题都被归类为 `quality_${type}`，丢失了原始维度信息。

#### 2. 问题描述缺乏具体定位

**用户困惑**：

- "场景描述过短" - 是哪个场景？还是所有场景都短？
- "3个分镜参数异常" - 具体是哪3个？什么参数异常？
- 无法快速定位到需要修复的具体对象

**现状代码**：

```typescript
// QualityAnalyzer.ts 第233-240行
if (incompleteScenes.length > 0) {
  issues.push({
    type: 'warning',
    message: `${incompleteScenes.length}个场景描述过短`, // 没有列出具体场景
    suggestion: '补充场景的环境、氛围、动作等细节描述',
    autoFixable: false,
  });
}
```

#### 3. 缺乏行业标准对比

**竞品分析**（来自`小说解析功能竞品分析报告.md`）：

| 产品          | 质量评估展示方式 | 特点                     |
| ------------- | ---------------- | ------------------------ |
| 白日梦AI      | 可视化仪表盘     | 进度条+具体数值+一键修复 |
| 即梦AI        | 卡片式问题列表   | 每个问题可展开查看详情   |
| 可灵AI        | 分级警告系统     | 严重/警告/提示三级分明   |
| Shai Creative | 专业报告PDF      | 可导出详细分析报告       |

**行业最佳实践**：

1. **问题可定位**：每个问题都能点击跳转到具体位置
2. **影响可视化**：用图表展示问题分布
3. **修复可操作**：每个问题都有明确的修复按钮或建议
4. **上下文丰富**：显示问题相关的上下文信息

## 优化方案

### 方案一：重构问题分类体系（高优先级）

#### 1.1 按维度分类展示问题

```typescript
// 新的问题分类结构
interface CategorizedIssues {
  completeness: QualityIssue[]; // 完整性问题
  accuracy: QualityIssue[]; // 准确性问题
  consistency: QualityIssue[]; // 一致性问题
  usability: QualityIssue[]; // 可用性问题
  dramatic: QualityIssue[]; // 戏剧性问题
}
```

#### 1.2 优化问题标签展示

**当前**：

```
🔴 4个警告  🔵 1个提示
```

**优化后**：

```
🔴 2个严重  🟡 3个警告  🔵 2个提示
├─ 完整性: 1警告
├─ 准确性: 1严重 1警告
├─ 一致性: 1警告
├─ 可用性: 1提示
└─ 戏剧性: 1提示
```

### 方案二：增强问题定位能力（高优先级）

#### 2.1 问题对象具体化

```typescript
// QualityAnalyzer.ts 修改
interface QualityIssue {
  type: 'error' | 'warning' | 'info';
  message: string;
  target?: string;           // 问题对象名称
  targetId?: string;         // 问题对象ID（用于跳转）
  targetType?: 'character' | 'scene' | 'shot' | 'item';  // 对象类型
  context?: string;          // 上下文信息
  suggestion: string;
  autoFixable?: boolean;
}

// 示例：场景描述过短问题
{
  type: 'warning',
  message: '场景描述过短',
  target: '场景1: 咖啡厅相遇',
  targetId: 'scene_001',
  targetType: 'scene',
  context: '当前描述: "咖啡厅内" (5字)，建议至少30字',
  suggestion: '补充场景的环境、氛围、动作等细节描述',
  autoFixable: false,
}
```

#### 2.2 问题列表可交互

## 回答用户的具体问题

### Q1: 4个警告是否对应优化建议的4点？

**现状**：

- 警告数量 = 所有 `type: 'warning'` 的问题总数
- 优化建议 = 基于低分维度和统计信息生成的文本建议
- **两者没有直接的一一对应关系**

**问题示例**：

```
警告列表：
1. [完整性] 3个角色缺少详细描述
2. [完整性] 2个场景描述过短
3. [一致性] 场景中出现未定义角色: 张三
4. [准确性] 3个分镜参数异常

优化建议：
1. [重要] 为角色补充外貌特征、性格描述等信息  ← 对应警告1
2. [建议] 补充场景的环境、氛围、动作等细节描述  ← 对应警告2
3. [建议] 检查分镜类型、描述、时长等参数      ← 对应警告4
4. [建议] 平均每场景分镜数较少，可考虑增加分镜丰富度  ← 基于统计，非警告
```

**优化后**：

```
问题与建议关联：
警告1 (3个角色) → 建议1 [修复]
警告2 (2个场景) → 建议2 [查看]
警告3 (1个角色) → 建议3 [添加角色]
警告4 (3个分镜) → 建议3 [检查]
```

### Q2: 提示是什么？

**提示的来源**：

```typescript
// QualityAnalyzer.ts 第454-459行 (analyzeUsability)
issues.push({
  type: 'info', // 这是提示类型
  message: `${unusableCount}个分镜描述较短`,
  suggestion: '补充分镜描述以提升生成质量',
  autoFixable: false,
});
```

**当前提示内容**：

- 分镜描述较短（可用性维度）
- 可能还有其他 `type: 'info'` 的问题

**问题**：提示被淹没在警告中，用户难以发现。

### Q3: "场景描述过短"指的是某个场景还是全部？

**现状**：

```typescript
// 只显示数量，不显示具体场景
message: `${incompleteScenes.length}个场景描述过短`;
```

**答案**：是指 **部分场景**（`incompleteScenes` 列表中的场景），不是所有场景。

**优化后**：

```
⚠️ 2个场景描述过短
├─ 场景1: 咖啡厅相遇 (当前5字，建议≥30字) [查看]
└─ 场景5: 办公室争吵 (当前8字，建议≥30字) [查看]
```

### Q4: "3个分镜参数异常"具体是哪3个？什么参数异常？

**现状**：

```typescript
// QualityAnalyzer.ts 第303-317行
const invalidShots = shots.filter(
  s => !s.type || !s.description || (s.duration && (s.duration < 1 || s.duration > 60))
);

issues.push({
  type: 'warning',
  message: `${invalidShots.length}个分镜参数异常`, // 没有具体信息
  suggestion: '检查分镜类型、描述、时长等参数',
  autoFixable: false,
});
```

**问题类型包括**：

- 缺少分镜类型 (`!s.type`)
- 缺少分镜描述 (`!s.description`)
- 时长异常 (`duration < 1 || duration > 60`)

**优化后**：

```
⚠️ 3个分镜参数异常
├─ 分镜#12: 缺少分镜类型 [编辑]
├─ 分镜#15: 时长异常 (当前: 0.5秒，范围: 1-60秒) [编辑]
└─ 分镜#23: 描述为空 [编辑]
```

### Q5: 行业前沿应用的质量评估面板展示形式

**竞品对比**（来自分析报告）：

| 产品              | 质量评估特点           | 与本项目差距     |
| ----------------- | ---------------------- | ---------------- |
| **白日梦AI**      | 仪表盘+进度条+一键修复 | 缺少一键修复功能 |
| **即梦AI**        | 卡片式+可展开详情      | 问题卡片不可展开 |
| **可灵AI**        | 三级警告+具体数值      | 缺少具体数值展示 |
| **Shai Creative** | 专业报告+PDF导出       | 缺少导出功能     |

**行业趋势**：

1. **从被动展示到主动修复**：不仅告诉用户有问题，还提供修复工具
2. **从汇总到明细**：从"5个警告"到"具体是哪5个"
3. **从静态到交互**：可点击、可跳转、可操作
4. **从文本到可视化**：图表、雷达图、热力图

**本项目现状 vs 行业**：

- 质量评估维度设计 ✅ 符合行业（5维度评分）
- 问题统计展示 ⚠️ 基础水平（只有数量）
- 问题定位能力 ❌ 落后（无法定位具体对象）
- 修复操作支持 ❌ 缺失（没有一键修复）
- 可视化展示 ❌ 缺失（没有图表）

## 总结

### 核心问题

1. **问题与建议脱节**：4个警告和4条建议没有明确对应关系
2. **定位信息缺失**：不知道"场景过短"是哪个场景，"分镜异常"是哪几个
3. **提示被淹没**：info级别的问题在UI上不够突出
4. **缺乏可操作性**：看到问题后无法直接跳转修复

### 优化优先级

| 优先级 | 优化项        | 预期效果                       |
| ------ | ------------- | ------------------------------ |
| 🔴 P0  | 问题具体定位  | 用户知道是哪个场景/角色/分镜   |
| 🔴 P0  | 问题-建议关联 | 清楚知道建议是为了解决什么问题 |
| 🟡 P1  | 问题分类展示  | 了解4个警告分别属于什么维度    |
| 🟡 P1  | 快速修复操作  | 一键跳转到编辑位置             |
| 🟢 P2  | 可视化图表    | 更直观的质量展示               |
| 🟢 P2  | 报告导出      | 符合行业最佳实践               |
