# 图片闪烁问题修复方案

## 问题深度分析

### 根本原因确认

经过深入分析，发现闪烁问题有多个触发源：

#### 1. 修改提示词时的闪烁

**原因**: `loadImageUrls` useEffect 依赖 `[allShots]`，修改提示词会更新 Script → allShots 变化 → 重新加载所有图片 URL

#### 2. 点击缩略图切换时的闪烁

**原因**:

- `handleSelectImage` 更新 Script 数据（修改 `currentImageId`）
- Script 更新 → `allShots` 变化 → `loadImageUrls` 重新执行
- 即使图片 URL 已经存在，也会经历"重置→重新获取"的过程

#### 3. 预览区闪烁

**原因**: 同上的机制，预览区的图片 URL 也被重置后重新获取

### 核心问题代码

```typescript
// 问题 1: 依赖 allShots 导致频繁触发
useEffect(() => {
  const loadImageUrls = async () => {
    const urls: Record<string, string> = {};

    // 遍历所有分镜和关键帧
    allShots.forEach(shot => {
      shot.keyframes?.forEach(kf => {
        if (kf.generatedImages) {
          kf.generatedImages.forEach(img => {
            if (img.path) {
              if (img.path.startsWith('http://') || img.path.startsWith('https://')) {
                urls[img.id] = img.path;
              } else {
                // 问题 2: 异步获取 URL，逐个更新
                storageService.getAssetUrl(img.path).then(url => {
                  setImageUrls(prev => ({ ...prev, [img.id]: url }));
                });
              }
            }
          });
        }
      });
    });

    // 问题 3: 先重置 imageUrls（只包含远程 URL）
    setImageUrls(urls);
  };

  if (allShots.length > 0) {
    loadImageUrls();
  }
}, [allShots]); // 依赖 allShots 导致频繁触发
```

### 为什么切换缩略图也会触发？

```typescript
const handleSelectImage = (imgId: string) => {
  // 更新 keyframe 的 currentImageId
  const updatedKeyframes = [...selectedShot.keyframes!];
  updatedKeyframes[selectedKeyframeIndex] = {
    ...kf,

  const updatedShot = { ...selectedShot, keyframes: updatedKeyframes };
  const updatedShots = allShots.map(s => (s.id === selectedShot.id ? updatedShot : s));
  const updatedScript = {
    ...currentScript!,
    parseState: {
      ...currentScript!.parseState,
      shots: updatedShots, // 新引用
    },
  };
  storageService.saveScript(updatedScript);
  setScripts(scripts.map(s => (s.id === updatedScript.id ? updatedScript : s)));
  // → allShots 变化 → loadImageUrls 触发 → 闪烁
};
```

## 修复方案

### 方案评估

| 方案                        | 优点           | 缺点                  | 适用性   |
| --------------------------- | -------------- | --------------------- | -------- |
| A. 依赖改为图片 path 列表   | 精确控制       | 实现复杂              | 中等     |
| B. 保留现有 URL，只加载新增 | 改动小，效果好 | 需要管理 URL 生命周期 | **推荐** |
| C. 使用 useMemo 缓存        | 简单           | 不能解决根本问题      | 低       |
| D. 使用防抖                 | 减少触发频率   | 延迟加载，体验差      | 低       |

### 推荐方案 B: 增量加载 + URL 缓存

核心思路：

1. **不清空现有 URL** - 只加载新增的图片
2. **使用图片 ID + path 作为依赖** - 只有真正新增图片时才加载
3. **组件卸载时清理 URL** - 避免内存泄漏

### 具体实现

#### 修改 1: 重写 loadImageUrls 逻辑

```typescript
// 提取所有图片的唯一标识（id + path）
const getImageIdentifiers = (shots: Shot[]): string[] => {
  const identifiers: string[] = [];
  shots.forEach(shot => {
    shot.keyframes?.forEach(kf => {
      if (kf.generatedImages) {
        kf.generatedImages.forEach(img => {
          if (img.path) {
            identifiers.push(`${img.id}:${img.path}`);
          }
        });
      }
      if (kf.generatedImage?.path) {
        identifiers.push(`${kf.generatedImage.id}:${kf.generatedImage.path}`);
      }
    });
  });
  return identifiers.sort(); // 排序确保比较一致性
};

// 使用图片标识作为依赖
const imageIdentifiers = useMemo(() => getImageIdentifiers(allShots), [allShots]);

useEffect(() => {
  const loadImageUrls = async () => {
    // 收集所有需要加载的图片
    const imagesToLoad: { id: string; path: string }[] = [];

    allShots.forEach(shot => {
      shot.keyframes?.forEach(kf => {
        if (kf.generatedImages) {
          kf.generatedImages.forEach(img => {
            if (img.path && !imageUrls[img.id]) {
              // 只加载还没有 URL 的图片
              if (img.path.startsWith('http://') || img.path.startsWith('https://')) {
                // 远程 URL 直接设置
                setImageUrls(prev => ({ ...prev, [img.id]: img.path }));
              } else {
                imagesToLoad.push({ id: img.id, path: img.path });
              }
            }
          });
        }
        if (kf.generatedImage?.path && !imageUrls[kf.generatedImage.id]) {
          if (
            kf.generatedImage.path.startsWith('http://') ||
            kf.generatedImage.path.startsWith('https://')
          ) {
            setImageUrls(prev => ({ ...prev, [kf.generatedImage.id]: kf.generatedImage.path }));
          } else {
            imagesToLoad.push({ id: kf.generatedImage.id, path: kf.generatedImage.path });
          }
        }
      });
    });

    // 异步加载本地图片 URL
    imagesToLoad.forEach(({ id, path }) => {
      storageService.getAssetUrl(path).then(url => {
        setImageUrls(prev => ({ ...prev, [id]: url }));
      });
    });
  };

  if (allShots.length > 0) {
    loadImageUrls();
  }
  // 依赖改为 imageIdentifiers，只有图片列表真正变化时才触发
}, [imageIdentifiers]);
```

#### 修改 2: 组件卸载时清理 URL（可选，用于内存管理）

```typescript
useEffect(() => {
  return () => {
    // 清理 blob URL，避免内存泄漏
    Object.values(imageUrls).forEach(url => {
      if (url.startsWith('blob:')) {
        URL.revokeObjectURL(url);
      }
    });
  };
}, []);
```

### 为什么这个方案最好？

1. **修改提示词时** - 只改变了 description/prompt，图片 path 不变 → `imageIdentifiers` 不变 → 不触发加载

2. **切换缩略图时** - 只改变了 `currentImageId`，图片 path 不变 → `imageIdentifiers` 不变 → 不触发加载

3. **生图完成时** - 新增了图片 → `imageIdentifiers` 变化 → 只加载新增的图片，已有图片 URL 保留

4. **保持现有功能** - 不需要修改 ThumbnailScroller 或其他组件，只改 useEffect 逻辑

## 备选方案：使用 useRef 缓存

如果希望更简单的实现，可以使用 useRef 来缓存 URL：

```typescript
const imageUrlsRef = useRef<Record<string, string>>({});
const [imageUrls, setImageUrls] = useState<Record<string, string>>({});

useEffect(() => {
  const loadImageUrls = async () => {
    const newUrls: Record<string, string> = {};

    allShots.forEach(shot => {
      shot.keyframes?.forEach(kf => {
        if (kf.generatedImages) {
          kf.generatedImages.forEach(img => {
            if (img.path) {
              // 优先使用缓存的 URL
              if (imageUrlsRef.current[img.id]) {
                newUrls[img.id] = imageUrlsRef.current[img.id];
              } else if (img.path.startsWith('http://') || img.path.startsWith('https://')) {
                newUrls[img.id] = img.path;
              } else {
                storageService.getAssetUrl(img.path).then(url => {
                  imageUrlsRef.current[img.id] = url;
                  setImageUrls(prev => ({ ...prev, [img.id]: url }));
                });
              }
            }
          });
        }
      });
    });

    // 只更新已缓存的 URL，不重置
    setImageUrls(prev => ({ ...prev, ...newUrls }));
  };

  if (allShots.length > 0) {
    loadImageUrls();
  }
}, [allShots]);
```

这个方案更简单，但依赖 allShots 还是会频繁触发，只是避免了闪烁。

## 最终推荐

**推荐方案 B（增量加载）**，因为它：

1. 从根本上解决问题（减少不必要的触发）
2. 保持项目现有功能（ThumbnailScroller 等组件无需修改）
3. 代码改动量适中
4. 性能最好（不重复加载已存在的 URL）
