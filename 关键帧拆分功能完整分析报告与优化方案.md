## 三、核心问题深度分析

### 3.1 生图提示词问题详解

**当前错误示例**：
```typescript
prompt: `参考角色图：kf_abc_1，参考场景图：scene_def；中景，客厅，张三，静态画面，电影级画质，开场画面，全景展示，光线柔和，建立氛围`
```

**问题点**：
1. ❌ 包含内部数据库ID：`kf_abc_1`、`scene_def` 对AI模型毫无意义
2. ❌ 中文逗号分隔：不符合AI生图提示词的标准格式（应该用英文逗号）
3. ❌ 缺少视觉细节：没有角色服装、表情、动作、光影、色彩等具体描述
4. ❌ 缺少质量控制：没有8K、超详细、最佳质量等质量标签
5. ❌ 风格不一致：没有融入全局视觉风格（artDirection、cinematography等）
6. ❌ 提示词与参考图脱节：提示词写"参考角色图"，但实际参考图是通过单独参数传递的

**应该有的提示词格式**：
```
masterpiece, 8k, ultra detailed, best quality, cinematic lighting, movie still, shot on 35mm film,
{artDirection}, {cinematography}, {lightingStyle}, {colorPalette},
{shotType}, {characterPose}, {characterExpression}, {characterClothing},
{sceneDescription}, {composition}, {lighting}, {colorMood},
{characterName} standing in {sceneName}, looking at camera with a serious expression, wearing a black suit,
reference to character design and scene setting
```

### 3.2 负面提示词问题详解

**数据流断裂**：
```
用户输入 negativePrompt (ShotManager.tsx:592)
    ↓
存储在状态中，但从未传递
    ↓
aiService.createKeyframeGenerationJob() 被调用 ❌ 缺少 negativePrompt
    ↓
provider.generateImage() 被调用 ❌ 接口没有 negativePrompt 参数
    ↓
API 请求发送 ❌ 没有负面提示词
```

**后果**：
- 无法避免低分辨率、模糊、变形等问题
- 用户在UI中设置的负面提示词完全无效
- 生图质量无法控制

### 3.3 全局视觉风格问题详解

**已有的数据结构**：
```typescript
interface VisualStyle {
  artDirection: string;           // 如：写实电影感、动漫风格、水墨国风
  artStyle: string;               // 艺术风格标签
  colorPalette: string[];          // 主色调
  colorMood: string;               // 如：温暖明亮、冷峻压抑、复古怀旧
  cinematography: string;          // 如：手持纪实、稳定器流畅、电影感构图
  lightingStyle: string;           // 如：自然光、戏剧光、noir风格
}
```

**完全未被使用**：
- KeyframeEngine 没有从 `currentScript.metadata.visualStyle` 读取
- 关键帧 prompt 中没有融入这些风格信息
- 同一剧本的不同关键帧可能风格迥异

### 3.4 参考图机制问题详解

**已有的资产特性**：
```typescript
CharacterAsset.views: {
  front?: GeneratedImage;      // 正面
  side?: GeneratedImage;       // 侧面
  back?: GeneratedImage;       // 背面
  threeQuarter?: GeneratedImage; // 四分之三侧面
}

SceneAsset.views: {
  panorama?: GeneratedImage;    // 全景图
  wide?: GeneratedImage;        // 广角图
  detail?: GeneratedImage[];   // 细节图数组
  aerial?: GeneratedImage;      // 鸟瞰图
}
```

**完全未被使用**：
- 关键帧拆分时没有根据关键帧的视角/景别选择对应的参考图
- 没有区分"角色正脸"用front图、"角色侧脸"用side图
- 没有区分"大远景"用panorama图、"近景"用detail图

## 四、优化增强方案

### 阶段一：短期修复（1-2周）

#### 4.1.1 重构关键帧提示词生成

**目标**：建立标准化的提示词结构

**实现要点**：
1. 创建 `KeyframePromptGenerator` 专门服务
2. 提示词结构分为：
   - **质量标签层**：masterpiece, 8k, ultra detailed, best quality
   - **风格层**：artDirection, cinematography, lightingStyle, colorPalette
   - **景别层**：shotType, cameraAngle
   - **角色层**：pose, expression, clothing, position
   - **场景层**：sceneDescription, composition, lighting, colorMood
   - **动作层**：具体的静态姿态描述
3. 使用英文逗号分隔，符合AI生图标准
4. 融入全局视觉风格

#### 4.1.2 修复负面提示词传递链路

**目标**：确保用户设置的负面提示词生效

**实现要点**：
1. 在 `Keyframe` 接口中添加 `negativePrompt` 字段
2. 在 `aiService.createKeyframeGenerationJob()` 中添加 `negativePrompt` 参数
3. 在所有 Provider 的 `generateImage()` 方法中添加 `negativePrompt` 参数并传递给API
4. 在关键帧拆分时，将用户设置的负面提示词保存到每个 Keyframe 中

#### 4.1.3 应用全局视觉风格

**目标**：确保同一剧本所有关键帧风格一致

**实现要点**：
1. 在 `KeyframeEngine.buildSplitPrompt()` 中读取 `currentScript.metadata.visualStyle`
2. 将视觉风格融入到LLM提示词中，要求LLM生成符合该风格的关键帧
3. 将视觉风格信息保存到每个 Keyframe 的结构化字段中
4. 在生成关键帧提示词时，自动融入这些风格信息

### 阶段二：中期优化（3-4周）

#### 4.2.1 重构 Keyframe 数据结构

**目标**：增加结构化视觉信息，便于一致性校验和编辑

**新的数据结构**：
```typescript
interface Keyframe {
  // 原有字段
  id: string;
  sequence: number;
  frameType: FrameType;
  status: 'pending' | 'generating' | 'completed' | 'failed';
  
  // === 新增：结构化视觉描述 ===
  visual: {
    // 角色描述（支持多个角色）
    characters: Array<{
      characterId: string;
      characterName: string;
      pose: string;           // 具体姿态，如"站立，右手插兜"
      expression: string;       // 表情，如"严肃，皱眉"
      clothing: string;         // 服装，确保一致性
      position: string;         // 在画面中的位置，如"画面左侧"
      viewAngle: 'front' | 'side' | 'back' | 'three-quarter'; // 视角
      gazeDirection: string;    // 视线方向
    }>;
    
    // 场景描述
    scene: {
      sceneId: string;
      sceneName: string;
      composition: string;      // 构图
      lighting: string;         // 光影
      colorPalette: string[];   // 色调
      focus: string;           // 焦点
    };
    
    // 道具描述
    items: Array<{
      itemId: string;
      itemName: string;
      position: string;
      state: string;
    }>;
    
    // 质量和风格
    quality: {
      resolution: string;
      style: string;
      artisticStyle: string;
      cinematography: string;
      lightingStyle: string;
    };
  };
  
  // === 新增：负面提示词 ===
  negativePrompt: string;
  
  // === 原有字段，重新设计 ===
  description: string;      // 人类可读的描述
  prompt: string;          // AI生图提示词（由visual自动生成）
  
  // === 参考图选择 ===
  referenceSelection: {
    characterViewAngle: 'front' | 'side' | 'back' | 'three-quarter';
    sceneViewType: 'panorama' | 'wide' | 'detail' | 'aerial';
  };
  
  // === 连贯性信息 ===
  continuity: {
    prevKeyframeId?: string;  // 前序关键帧ID
    nextKeyframeId?: string;  // 后序关键帧ID
    continuityNotes?: string;  // 连贯性注意事项
  };
  
  // 原有生成字段
  duration: number;
  references: { ... };
  generatedImage?: GeneratedImage;
  generatedImages?: GeneratedImage[];
  currentImageId?: string;
}
```

#### 4.2.2 利用三视图和多视角

**目标**：根据关键帧视角选择正确的参考图

**实现要点**：
1. 在LLM提示词中，要求LLM为每个关键帧推荐角色视角（front/side/back/three-quarter）
2. 根据推荐的视角，从 `CharacterAsset.views` 中选择对应的参考图
3. 根据景别，从 `SceneAsset.views` 中选择对应的参考图（大远景用panorama，近景用detail）
4. 将选择的参考图信息保存到 `Keyframe.referenceSelection`

#### 4.2.3 增强LLM提示词工程

**目标**：提升LLM输出质量和一致性

**实现要点**：
1. 添加 Few-shot 示例：在LLM提示词中提供2-3个"好的关键帧描述"示例
2. 添加强一致性约束：反复强调"同一角色在不同关键帧中服装、发型必须完全一致"
3. 添加视觉细节要求：要求描述"角色的右手放在哪里"、"眼神看向何方"等具体细节
4. 添加前后关键帧对比：对于middle和end帧，要求与start帧进行对比，描述变化
5. 建立输出Schema验证：使用JSON Schema验证LLM输出

#### 4.2.4 集成 PromptGeneratorService

**目标**：复用已有的提示词生成服务

**实现要点**：
1. 在关键帧拆分完成后，使用 `PromptGeneratorService` 生成最终的生图提示词
2. 配置 `PromptGeneratorService` 开启质量标签和负面提示词
3. 将结构化的 `Keyframe.visual` 数据传递给 `PromptGeneratorService`

### 阶段三：长期完善（5-8周）

#### 4.3.1 实现分镜间连贯性

**目标**：确保前序分镜尾帧与后序分镜首帧连贯

**实现要点**：
1. 拆分关键帧时，同时考虑前序分镜的最后一个关键帧
2. 建立"角色状态追踪"：记录角色在分镜结束时的位置、朝向、表情
3. 将前序分镜的尾帧信息传递给当前分镜的首帧生成
4. 添加连贯性验证：检查角色在分镜间的状态变化是否合理

#### 4.3.2 添加关键帧预览与编辑

**目标**：让用户可以查看和编辑关键帧的视觉描述

**实现要点**：
1. 在UI中展示结构化的 `Keyframe.visual` 数据
2. 允许用户编辑角色姿态、表情、服装等
3. 允许用户切换角色视角（front/side/back）
4. 实时重新生成提示词
5. 添加"一键应用风格到所有关键帧"功能

#### 4.3.3 建立一致性校验系统

**目标**：自动检测和警告一致性问题

**实现要点**：
1. 角色一致性校验：检查同一角色在不同关键帧中的服装、发型是否一致
2. 场景一致性校验：检查同一场景在不同关键帧中的色调、光影是否一致
3. 连贯性校验：检查分镜间的角色状态变化是否合理
4. 提供修复建议：发现问题时，提供自动修复或手动修复选项

#### 4.3.4 建立提示词模板库

**目标**：根据不同风格、场景提供预设提示词模板

**实现要点**：
1. 按艺术风格分类：写实电影感、动漫风格、水墨国风、赛博朋克等
2. 按摄影风格分类：手持纪实、稳定器流畅、电影感构图等
3. 按场景类型分类：室内、室外、日景、夜景等
4. 允许用户保存和管理自定义模板

## 六、总结

当前关键帧拆分功能与最初的设计目标存在**巨大差距**：

| 设计目标 | 当前现状 | 差距 |
|---------|---------|------|
| ✅ 确保视频一致性 | ❌ 关键帧间无连贯性，风格不一致 | 严重 |
| ✅ 基于角色/场景资产 | ❌ 三视图、多视角完全未利用 | 严重 |
| ✅ 生成高质量生图提示词 | ❌ 提示词极度不规范，包含内部ID | 致命 |
| ✅ 参考图+提示词保障一致性 | ❌ 提示词与参考图脱节，负面提示词未使用 | 严重 |
| ✅ 作为视频生成首尾帧参考 | ❌ 关键帧质量无法保证，无法用于视频生成 | 致命 |

**建议立即开始阶段一的修复工作**，特别是：
1. 修复负面提示词传递（0.5天）
2. 重构提示词生成（2天）
3. 应用全局视觉风格（1天）

这些工作可以在1-2周内完成，立即显著提升关键帧拆分功能的质量。
