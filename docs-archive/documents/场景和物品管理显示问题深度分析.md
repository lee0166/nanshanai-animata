# 场景和物品管理显示问题深度分析

## 问题汇总

从用户截图分析：

### 1. 角色管理 ✅ 正常

- 筛选器工作正常
- 左侧卡片和右侧列表同步显示

### 2. 场景管理 ❌ 异常

- 显示 "NO SCENES FOUND"
- 用户确认已创建场景，但不显示
- 筛选器选择了"初夏"剧本

### 3. 物品管理 ❌ 异常

- 显示两个筛选器：
  - 标题行："全部剧本"
  - 中间行："物品类型" + "全部"
- 显示 "NO ITEMS FOUND"

## 深度分析

### 场景管理问题

**可能原因1：筛选逻辑错误**

- 选择了"初夏"剧本，但场景可能没有关联此剧本
- 筛选条件过于严格，导致没有匹配项

**可能原因2：数据未正确保存**

- 创建场景时 `scriptId` 未正确保存
- 或者保存后数据格式不正确

**可能原因3：AssetList 筛选逻辑问题**

- 场景筛选逻辑可能与角色不同
- 需要检查 AssetList.tsx 中的场景筛选代码

### 物品管理问题

**问题1：重复筛选器**

- 标题行有一个"全部剧本"筛选器
- 中间还有一个"物品类型"筛选区域
- 看起来有两个筛选器UI

**问题2：不显示物品**

- 可能和场景管理同样的问题
- 或者物品筛选逻辑有问题

## 诊断方案

### 步骤1：检查 AssetList 的筛选逻辑

检查 AssetList.tsx 中的场景和物品筛选代码：

```typescript
// 检查场景筛选
if (
  (type === AssetType.CHARACTER || type === AssetType.SCENE || type === AssetType.ITEM) &&
  externalScriptFilter &&
  externalScriptFilter !== 'all'
) {
  // 筛选逻辑
}
```

### 步骤2：检查创建时 scriptId 保存

检查创建场景/物品时是否正确保存了 `scriptId`

### 步骤3：检查物品管理的重复筛选器

检查为什么物品管理有两个筛选器UI

### 步骤4：添加调试日志

在关键位置添加调试，查看：

- 加载的资产数据
- 筛选条件
- 筛选后的结果

## 修复方案

### 修复1：场景/物品筛选逻辑

确保 AssetList 的场景和物品筛选逻辑正确

### 修复2：移除物品管理的重复筛选器

如果 AssetList 已经显示筛选器，ProjectDetail 不应该再显示

### 修复3：添加调试信息

添加日志查看实际数据，帮助诊断问题
