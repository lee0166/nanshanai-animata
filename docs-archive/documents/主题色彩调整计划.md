# 优化剧本筛选器位置

## 问题描述

当前剧本筛选器区域太大（占了一整行），用户希望将其移到"角色管理"标题右侧，使布局更紧凑。

## 当前布局

```
┌─────────────────────────────────────────────────────────────┐
│  角色管理                                          [新建角色] │
├─────────────────────────────────────────────────────────────┤
│  ┌───────────────────────────────────────────────────────┐  │
│  │  剧本筛选  [全部剧本 ▼]                               │  │  ← 占一整行
│  └───────────────────────────────────────────────────────┘  │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────┐ ┌─────────┐                                    │
│  │ 角色A   │ │ 角色B   │                                    │
```

## 目标布局

```
┌─────────────────────────────────────────────────────────────┐
│  角色管理  [全部剧本 ▼]                            [新建角色] │  ← 同一行
├─────────────────────────────────────────────────────────────┤
│  ┌─────────┐ ┌─────────┐                                    │
│  │ 角色A   │ │ 角色B   │                                    │
```

## 实现方案

### 方案：调整布局结构

**步骤1：在 ProjectDetail.tsx 标题行添加筛选器**

- 在"角色管理"标题旁边添加剧本筛选器

- 只在角色类型页面显示

**步骤2：从 AssetList.tsx 移除筛选器区域**

- 移除当前的大筛选器UI

- 保留筛选逻辑（根据外部传入的 scriptFilter）

### 修改文件

1. **views/ProjectDetail.tsx**
   - 在标题区域添加剧本筛选器

   - 保持筛选状态管理

2. **components/AssetList.tsx**
   - 移除剧本筛选器UI

   - 保留按剧本筛选的逻辑

### 复杂度评估

| 任务                    | 复杂度 | 说明                 |
| ----------------------- | ------ | -------------------- |
| 移动筛选器到标题行      | 低     | 只是UI位置调整       |
| 从 AssetList 移除筛选器 | 低     | 删除UI代码           |
| 保持筛选功能            | 低     | 逻辑已存在，无需修改 |

**总体复杂度：低**

### 预期效果

- 布局更紧凑，筛选器与标题同行

- 功能保持不变

- 只在角色页面显示筛选器
