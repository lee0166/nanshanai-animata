# 角色图片上传功能需求分析计划

## 需求理解

### 用户场景

1. 用户在创建角色并生成角色图片
2. 对AI生成的结果不满意
3. 手上有其他渠道获得的满意图片（如自己绘制、从其他平台获取等）
4. 希望将这些图片上传到项目中，关联到对应角色
5. **当前缺失此功能**

### 核心需求

- 在角色生成结果页面，增加上传本地图片的功能
- 上传的图片应能像AI生成的图片一样被保存和管理
- 支持后续基于此图片进行视频生成等操作

### UI/UX 需求（根据用户截图）

- **上传按钮位置**：在"生成结果"标题右侧，"图片 (2)" 文字的左侧（红色框区域）
- **展示方式**：上传后的图片应与AI生成的图片一样，以相同的卡片形式展示在结果区域
- **交互体验**：与现有生图结果保持一致的体验

---

## 调研内容

### 1. 角色生成结果的保存方式

需要调研：

- 角色图片的存储位置（本地文件系统）
- 图片的命名规则
- 图片的元数据如何记录（数据库/JSON文件）
- 图片与角色的关联关系

### 2. 现有图片生成流程

需要调研：

- AI生成图片后如何保存
- 图片如何展示在角色详情页
- 图片的预览、选择、删除逻辑

### 3. 文件上传相关

需要调研：

- 项目是否已有文件上传组件
- 本地文件系统的访问方式（File System Access API）
- 图片格式和大小的限制

### 4. UI组件分析

需要调研：

- "生成结果"区域的组件结构
- 图片卡片组件的实现
- 如何在标题栏添加上传按钮

---

## 调研步骤

### 步骤一：查看角色相关组件

- `components/ProjectDetail/Character/` 目录下的组件
- 角色详情页、角色卡片、图片生成面板等
- 重点查看"生成结果"区域的实现

### 步骤二：查看图片存储服务

- `services/storage.ts` - 存储服务
- 图片保存、读取、删除的方法

### 步骤三：查看图片生成流程

- 图片生成后的保存逻辑
- 元数据记录方式

### 步骤四：查看文件上传相关

- 是否有现有的文件上传组件
- File System Access API 的使用方式

### 步骤五：分析实现方案

基于调研结果，设计：

1. 数据存储方案
2. UI组件方案（在生成结果标题栏添加上传按钮）
3. 上传流程
4. 图片展示方案（与AI生成图片一致）
