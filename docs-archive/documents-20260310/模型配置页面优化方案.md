# 模型配置页面优化方案

## 项目概述

基于 UI-UX-Pro-Max Skill 的 UX 准则分析，针对 AI Animata 设置页面的模型配置模块进行全面优化。

## 当前问题诊断

### 🔴 高优先级问题

#### 1. 搜索和过滤功能缺失 (UX 准则 #89, #90)

- **问题**: 12+ 个模型平铺展示，无法快速定位目标模型
- **影响**: 用户效率极低，需要大量滚动查找
- **解决方案**:
  - 添加搜索框（支持名称/Model ID 模糊搜索）
  - 添加类型过滤器（图像/视频/文本）
  - 添加提供商过滤器
  - 添加排序功能（名称/类型/最近使用）

#### 2. API Key 输入框无显示/隐藏切换 (UX 准则 #60)

- **问题**: 密码输入框无法查看内容，用户无法确认输入是否正确
- **影响**: 可用性差，容易输入错误
- **解决方案**:
  - 添加 Eye/EyeOff 图标按钮切换显示/隐藏
  - 点击图标切换 input type between "password" 和 "text"

#### 3. 操作按钮移动端不可见 (UX 准则 #28, #40)

- **问题**: 编辑/删除按钮使用 group-hover 显示，移动端无法悬停
- **影响**: 移动端用户无法发现操作按钮
- **解决方案**:
  - 移动端始终显示操作按钮（使用媒体查询）
  - 桌面端保持悬停显示，但添加过渡动画
  - 为按钮添加 aria-label 无障碍属性

### 🟡 中优先级问题

#### 4. 信息密度过高 (UX 准则 #21, #76)

- **问题**: 所有模型平铺，每个卡片包含过多信息
- **影响**: 视觉疲劳，难以快速扫描
- **解决方案**:
  - 按模型类型分组显示（图像生成/视频生成/文本解析）
  - 使用 Accordion 手风琴组件折叠/展开
  - Model ID 默认折叠，点击展开
  - 增加卡片间距

#### 5. 缺少批量操作功能 (UX 准则 #91)

- **问题**: 只能一个一个编辑/删除模型
- **影响**: 管理多个模型效率低下
- **解决方案**:
  - 添加复选框选择多个模型
  - 添加批量操作栏（删除选中/测试连接/导出配置）
  - 添加"全选"功能

#### 6. 状态反馈不足 (UX 准则 #78, #82)

- **问题**: 不知道 API Key 是否有效，模型是否可用
- **影响**: 用户不确定性，可能在无效模型上浪费时间
- **解决方案**:
  - 添加"测试连接"按钮验证 API Key
  - 显示连接状态指示器（绿色正常/红色失效/灰色未测试）
  - 显示最后测试时间

### 🟢 低优先级问题

#### 7. 视觉层次优化

- **问题**: 卡片背景与页面背景对比度不足
- **解决方案**:
  - 调整卡片背景色为稍亮的 slate-800/50
  - 添加细微边框增强边界感
  - 优化类型标签颜色对比度

#### 8. 空 API Key 提示

- **问题**: API Key 为空时无明显视觉提示
- **解决方案**:
  - API Key 为空时显示警告图标
  - 添加"未配置"标签
  - 输入框边框变红提示

## 技术实现方案

### 组件结构变更

```
ModelManager (模型管理主组件)
├── SearchAndFilterBar (搜索过滤栏)
│   ├── SearchInput (搜索输入框)
│   ├── TypeFilter (类型过滤器)
│   ├── ProviderFilter (提供商过滤器)
│   └── SortSelector (排序选择器)
├── BulkActionBar (批量操作栏) - 选中时显示
│   ├── SelectAllCheckbox (全选)
│   ├── TestSelectedButton (测试选中)
│   └── DeleteSelectedButton (删除选中)
└── ModelGroups (模型分组列表)
    └── ModelGroup (模型分组 - 按类型)
        ├── GroupHeader (分组标题 + 展开/折叠)
        └── ModelCards (模型卡片列表)
            └── ModelCard (单个模型卡片)
                ├── CardHeader (名称 + 类型标签 + 状态)
                ├── CardBody (Model ID + API Key)
                │   └── ApiKeyInput (带显示/隐藏切换)
                └── CardActions (编辑/删除按钮)
```

## 验收标准

### 功能验收

- [ ] 搜索功能可以按名称/Model ID 过滤模型
- [ ] 过滤器可以多选类型和提供商
- [ ] API Key 输入框可以切换显示/隐藏
- [ ] 移动端可以正常看到操作按钮
- [ ] 模型可以按类型分组展开/折叠
- [ ] 可以批量选择和操作多个模型
- [ ] 可以测试模型连接状态

### 无障碍验收

- [ ] 所有按钮有 aria-label
- [ ] 颜色不是唯一的状态指示方式
- [ ] 键盘可以操作所有功能

### 视觉验收

- [ ] 卡片有清晰的边界
- [ ] 标签颜色对比度符合 WCAG AA 标准
- [ ] 悬停和焦点状态可见

## 风险评估

| 风险              | 可能性 | 影响 | 缓解措施           |
| ----------------- | ------ | ---- | ------------------ |
| 批量操作误删      | 中     | 高   | 添加确认对话框     |
| 测试连接 API 超时 | 高     | 中   | 添加超时处理和重试 |
| 移动端布局问题    | 中     | 中   | 充分测试各尺寸     |
