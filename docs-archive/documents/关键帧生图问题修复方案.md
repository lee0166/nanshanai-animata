# 关键帧生图问题修复方案

## 问题分析

### 问题 1: 生图完成后预览区不更新

**原因**: ShotManager 没有订阅任务队列的更新。任务完成后虽然 Script 数据已更新，但组件状态没有重新加载。

**解决方案**:

1. 在 ShotManager 中添加对 jobQueue 的订阅
2. 当关键帧生图任务完成时，重新加载对应的 Script 数据

### 问题 2: 生图过程中按钮不是不可点击状态

**原因**: 使用 `keyframe.status === 'generating'` 判断，但任务提交后没有设置 keyframe.status 为 'generating'。

**解决方案**:

1. 在提交任务时立即更新 keyframe.status 为 'generating'
2. 或者使用任务队列的状态来判断（推荐）

### 问题 3: 生图完成后无提示

**原因**: 任务完成后没有显示 Toast 提示。

**解决方案**:

1. 在订阅回调中检测任务完成状态
2. 显示成功或失败的 Toast 提示

## 修复方案

### 修改文件: views/ShotManager.tsx

#### 修复 1: 添加任务队列订阅

在 useEffect 中添加订阅：

```typescript
// 在文件顶部导入
import { jobQueue } from '../services/queue';
import { Job, JobStatus } from '../types';

// 在组件内部添加订阅 useEffect
useEffect(() => {
  // 订阅任务队列更新
  const unsubscribe = jobQueue.subscribe(async (job: Job) => {
    // 只处理关键帧生图任务
    if (job.type !== 'generate_keyframe_image') return;

    // 检查是否属于当前项目的任务
    if (job.projectId !== projectId) return;

    // 检查任务是否完成或失败
    if (job.status === JobStatus.COMPLETED || job.status === JobStatus.FAILED) {
      // 重新加载脚本数据
      const updatedScripts = await storageService.getScripts(projectId);
      setScripts(updatedScripts);

      // 显示提示
      if (job.status === JobStatus.COMPLETED) {
        showToast('关键帧图片生成成功', 'success');
      } else {
        showToast(`生成失败: ${job.error || '未知错误'}`, 'error');
      }
    }
  });

  return () => {
    unsubscribe();
  };
}, [projectId]);
```

#### 修复 2: 提交任务时设置 keyframe status

修改 handleGenerateImage 方法：


#### 修复 3: 使用任务队列状态判断按钮 loading

替代方案（更精确）：使用 jobQueue 的活跃任务来判断

```typescript
// 添加状态来跟踪进行中的关键帧任务
const [activeKeyframeJobs, setActiveKeyframeJobs] = useState<Set<string>>(new Set());

useEffect(() => {
  const unsubscribe = jobQueue.subscribe((job: Job) => {
    if (job.type !== 'generate_keyframe_image') return;
    if (job.projectId !== projectId) return;

    setActiveKeyframeJobs(prev => {
      const next = new Set(prev);
      if (job.status === JobStatus.PROCESSING || job.status === JobStatus.PENDING) {
        next.add(job.params.keyframeId);
      } else {
        next.delete(job.params.keyframeId);
      }
      return next;
    });

    // 完成时刷新数据...
  });

  return () => unsubscribe();
}, [projectId]);

// 按钮 isLoading 判断
isLoading={activeKeyframeJobs.has(selectedShot.keyframes[selectedKeyframeIndex].id)}
```

## 推荐的简化方案

为了保持代码简洁，推荐以下方案：

1. **订阅任务队列** - 任务完成时自动刷新 Script 数据并显示 Toast
2. **提交时设置 generating 状态** - 立即更新 UI，让用户知道任务已提交
3. **使用 keyframe.status 判断 loading** - 保持现有逻辑，但确保状态正确设置

这样修改量最小，且能解决问题。
