# 三个问题修复计划

## 问题1：文本解析进度显示多位小数

### 问题描述

文本解析时进度会显示小数点后多位数字，需要限制为两位小数。

### 可能位置

- 剧本解析进度显示组件
- 进度计算逻辑

### 修复方案

在显示进度时，使用 `toFixed(2)` 限制小数位数。

---

## 问题2：角色管理页面未按剧本区分角色

### 问题描述

在项目中有多个剧本时，角色管理页面显示所有剧本的角色混在一起，没有按剧本区分。

### 需求

- 添加选项框："全部显示" / "按剧本显示"
- 选择"按剧本显示"时，可以筛选特定剧本的角色

### 可能位置

- 角色管理页面组件
- 角色数据获取和过滤逻辑

### 修复方案

1. 在角色管理页面添加筛选器
2. 角色数据添加剧本ID关联
3. 实现按剧本筛选功能

---

## 问题3：创建项目后角色创建失败且解析结果丢失

### 问题描述

1. 创建新项目（如 test 002）
2. 上传剧本并解析成功
3. 选择角色标签，在单个角色卡片中创建角色
4. 提示错误、失败
5. 刷新页面后，之前的解析结果消失，需要重新解析

### 可能原因

1. 角色创建时数据未正确保存
2. 剧本解析结果未持久化存储
3. 页面状态管理问题

### 修复方案

1. 检查角色创建的数据流和保存逻辑
2. 确保剧本解析结果正确存储到数据库/文件
3. 检查页面刷新后数据加载逻辑

---

## 实施顺序

建议按以下顺序修复：

1. **问题1**（最简单，纯UI修复）
2. **问题3**（影响核心功能，数据丢失严重）
3. **问题2**（功能增强，相对独立）
