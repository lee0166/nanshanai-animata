## 项目简介

**NS AI Animata** 是一款专为影视创作爱好者打造的 AI 驱动的小说转影视全流程工具（本地优先 Local-First），从小说文本到影视作品一站式生成，能力持续进化，既友好新手也支持深度自定义，与创作者同步学习、成长。

- 🏠 **完全本地运行** - 无需后端服务器，数据存储在本地文件系统
- 🎭 **智能剧本解析** - AI 自动提取角色、场景、分镜，支持进度追踪和后台运行
- 🎨 **多风格角色设计** - 8 种预设风格，支持角色一致性控制
- 🎬 **关键帧工作流** - 分镜自动拆分为关键帧，支持参考图生图
- 🎵 **音频管理系统** - 对话/音效/音乐分类管理，支持批量生成
- 📊 **剧本深度分析** - 故事结构、情感弧线、视觉风格自动识别
- 🤖 **多 AI 提供商** - 支持火山引擎、Vidu、ModelScope，可自定义模型
- ⚡ **高效任务队列** - 并发控制、失败重试、实时状态监控

## 技术架构

### 核心技术栈

**前端框架**：React 19.2.3 + TypeScript 5.8.2 + Vite 6.2.0\
**UI 组件**：HeroUI 2.8.7 + Tailwind CSS 4.1.18\
**动画库**：Framer Motion 12.23.26\
**图标库**：Lucide React 0.562.0\
**文档解析**：mammoth (Word) + pdfjs-dist (PDF)\
**AI/ML**：@xenova/transformers + ChromaDB (本地文本嵌入)\
**测试工具**：Vitest + @testing-library/react\
**开发工具**：ESLint + Prettier + Husky + Commitlint

### Local-First 架构

项目采用完全本地优先的架构，通过 File System Access API 直接读写本地文件系统，无需后端服务器，数据完全存储在本地，保障隐私安全。

```
浏览器 (Chrome/Edge)
  └─ React SPA
      ├─ 视图层
      ├─ 组件层
      └─ 服务层
         ↓
  File System Access API
         ↓
  本地文件系统 (projects/、assets/、scripts/)
```

### 服务层架构

前端实现了完整的服务化架构，核心模块如下：

- **AI 服务模块**：多提供商适配（火山引擎、Vidu、ModelScope、OpenAI 兼容）、智能路由、模型配置管理、健康检查、能力管理、插件化架构
- **剧本解析引擎**：五阶段智能解析（元数据→角色→场景→物品→分镜）、质量评估、一致性检查、迭代优化、语义分块、性能监控
- **关键帧服务**：分镜自动拆分为关键帧、参考图生图
- **视频生成服务**：图生视频、文生视频、运镜参数调节
- **资产管理服务**：自动归档、元数据追踪、资产复用
- **多层缓存系统**：Token 优化（节省约 80%）、语义向量缓存
- **任务队列系统**：并发控制、失败重试、实时状态监控
- **审核服务**：分镜质量检查、资产审核流程
- **编辑服务**：时间线编辑、转场效果

## 快速开始使用指南

### 1. 首次启动 - 连接工作区

1. 启动应用后，会看到欢迎页面
2. 点击 **"打开本地文件夹"** 按钮
3. 选择一个空文件夹作为您的工作区（所有项目和资产将存储在这里）
4. 授权浏览器访问该文件夹

### 2. 创建新项目

1. 在仪表盘页面点击 **"新建项目"**
2. 输入项目名称和描述
3. 点击 **"创建"** 完成项目创建

### 3. 导入剧本

1. 进入项目详情页，点击左侧 **"剧本"** 标签
2. 点击 **"导入剧本"** 按钮
3. 选择您的剧本文件（支持 TXT / DOCX / PDF 格式）
4. 或直接在编辑器中粘贴剧本内容

### 4. 配置创作意图

1. 导入剧本后，在剧本管理页面配置 **创作意图**
2. 选择影视风格：短剧/电影/纪录片/自定义
3. 设置叙事重点（主角成长、情感核心、世界观构建等）
4. 配置情感基调和视觉参考

### 5. 智能解析剧本

1. 配置完成后，点击 **"开始解析"**
2. 系统会自动执行以下阶段：
   - **阶段 1**: 提取故事元数据（标题、字数、角色列表等）
   - **阶段 2**: 识别并提取角色信息
   - **阶段 3**: 提取场景描述
   - **阶段 4**: 识别关键物品
   - **阶段 5**: 生成分镜列表
3. 解析过程中可以看到实时进度条
4. 解析完成后会生成质量评估报告

### 6. 查看剧本分析报告

1. 解析完成后，查看 **剧本分析** 页面
2. 查看故事结构图（三幕结构、情节点分布）
3. 查看情感弧线可视化
4. 查看自动识别的视觉风格
5. 查看故事核心要素摘要

### 7. 生成角色设计

1. 点击左侧 **"角色"** 标签
2. 选择一个解析出的角色，或点击 **"新建角色"**
3. 填写角色名称和描述
4. 选择美术风格（8 种预设风格可选）
5. 配置生成参数（分辨率、数量等）
6. 点击 **"生成"** 按钮
7. 在历史记录中浏览生成的图片，选择满意的一张设为当前
8. （可选）切换到其他视角（侧面/背面/四分之三）生成三视图

### 8. 生成场景设计

1. 点击左侧 **"场景"** 标签
2. 选择或创建场景
3. 配置风格和参数（与角色生成类似）
4. 点击 **"生成"** 创建场景图
5. （可选）生成多视角场景图（全景/广角/细节/鸟瞰）

### 9. 分镜管理与关键帧生成

1. 点击左侧 **"分镜"** 标签，或进入 **"分镜管理"** 页面
2. 查看按场景分组的分镜列表
3. 点击某个分镜进入详情
4. 系统会自动将分镜拆分为 2-3 个关键帧
5. 为每个关键帧生成图片：
   - 可以使用文生图模式
   - 也可以关联已生成的角色/场景作为参考图
6. 历史图片可以横向滚动浏览和切换

### 10. 视频生成

1. 在分镜详情页，关键帧图片生成完成后
2. 点击 **"生成视频"** 按钮
3. 配置视频参数：
   - 选择视频模型
   - 设置运镜方式（推/拉/摇/移等）
   - 配置时长
4. 点击开始生成
5. 视频生成任务会在后台队列中运行

### 11. 音频管理

1. 进入 **音视频管理** 页面
2. 选择音频类型（对话/音效/音乐）
3. 搜索或批量生成音频
4. 预览音频效果

### 12. 时间线编辑

1. 进入 **时间线编辑器** 页面
2. 将视频片段和音频拖拽到对应的轨道
3. 调整片段顺序和时长
4. 配置转场效果和音量

### 13. 审核管理

1. 进入 **审核管理** 页面
2. 查看待审核的分镜和资产
3. 进行质量检查
4. 标记为通过或拒绝

### 14. 模型配置

1. 点击右上角设置图标进入 **"设置"** 页面
2. 在 **"模型配置"** 部分：
   - 添加新的模型提供商
   - 配置 API 密钥
   - 设置默认模型
   - 调整高级参数（Temperature、Max Tokens 等）
3. 支持的提供商：
   - 火山引擎（Doubao、SeedEdit、HiDream）
   - Vidu
   - ModelScope
   - OpenAI 协议兼容的自定义模型

### 15. 任务监控

1. 右下角的 **"任务监控"** 面板会显示所有后台任务
2. 可以查看任务状态（等待中/处理中/已完成/失败）
3. 支持展开/折叠/最小化三种视图模式
4. 所有任务完成后会自动最小化

## 常见问题

### Q: 为什么需要选择本地文件夹？

A: Nanshan AI Animata 采用 Local-First 架构，通过 File System Access API 直接读写本地文件，确保您的所有数据（包括项目文件、生成的资产、API 密钥）都存储在本地，不会上传到任何云端服务器，最大程度保障数据隐私。

### Q: 支持哪些 AI 提供商？

A: 目前已集成火山引擎（Doubao/SeedEdit/HiDream）、Vidu、ModelScope、阿里云视频等。同时支持 OpenAI 协议兼容的自定义模型。

### Q: API 密钥安全吗？

A: API 密钥仅存储在您的浏览器本地存储（LocalStorage）中，不会被上传到任何服务器。您可以随时在设置页面删除密钥。

### Q: 如何贡献模型配置？

A: 在设置页面的模型配置部分，您可以添加、编辑、删除任意模型配置。配置会保存在本地，也可以导出分享。

### Q: 支持哪些浏览器？

A: 推荐使用 Chrome 120+ 或 Edge 120+，因为需要 File System Access API 支持。Firefox 和 Safari 目前不支持该 API。
