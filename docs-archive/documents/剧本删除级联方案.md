# 剧本删除级联删除方案分析

## 一、问题现状

### 1.1 问题描述

- 用户删除了两个剧本
- 剧本管理页面显示没有剧本（正确）
- 但角色管理、场景管理、物品管理、资源管理页面仍然显示已删除剧本的相关资源（错误）

### 1.2 问题根因

**代码位置**: `services/storage.ts` 第1125-1132行

```typescript
async deleteScript(scriptId: string, projectId: string): Promise<void> {
    const filename = `scripts_${projectId}.json`;
    return this.lock(filename, async () => {
      let scripts = await this.getScripts(projectId);
      scripts = scripts.filter(s => s.id !== scriptId);
      await this.writeJson(filename, scripts);
    });
  }
```

**问题**: `deleteScript` 方法**只删除了剧本本身**，没有级联删除与该剧本关联的角色、场景、物品等资产。

### 1.3 数据关联关系

根据 `types.ts` 中的定义，资产通过 `scriptId` 字段关联剧本：

```typescript
export interface Asset {
  id: string;
  projectId: string;
  scriptId?: string; // 关联剧本ID
  type: AssetType;
  // ...
}
```

涉及的资产类型：

- `CharacterAsset` - 角色
- `SceneAsset` - 场景
- `ItemAsset` - 物品

## 二、级联删除方案分析

### 2.1 方案一：硬删除（级联删除）

**实现方式**: 删除剧本时，同时删除所有关联的角色、场景、物品及其资源文件。

#### 优势

| 优势             | 说明                                                         |
| ---------------- | ------------------------------------------------------------ |
| **数据一致性**   | 删除剧本后，关联资源不会成为"孤儿数据"，数据关系始终保持一致 |
| **存储空间回收** | 及时释放被删除剧本占用的存储空间（图片、视频等文件）         |
| **用户体验清晰** | 用户删除剧本时期望其所有相关内容都被删除，符合直觉           |
| **避免数据污染** | 防止已删除剧本的资源在其他地方被误用或显示                   |
| **简化查询逻辑** | 不需要在查询时额外过滤已删除剧本的孤儿资源                   |

#### 劣势

| 劣势             | 说明                                                                       |
| ---------------- | -------------------------------------------------------------------------- |
| **误删风险**     | 用户误删剧本时，所有关联资源将永久丢失，无法恢复                           |
| **共享资源问题** | 如果角色/场景被多个剧本共享（通过相同assetId引用），级联删除会影响其他剧本 |
| **操作耗时**     | 如果剧本包含大量资源，删除操作可能需要较长时间，需要异步处理或加载提示     |
| **不可逆性**     | 删除后无法撤销，没有"回收站"机制                                           |

---

### 2.2 方案二：软删除（标记删除）

**实现方式**: 添加 `isDeleted` 标记，实际数据保留，仅在查询时过滤。

#### 优势

| 优势         | 说明                                         |
| ------------ | -------------------------------------------- |
| **可恢复性** | 用户误删后可以恢复，提供"回收站"或"撤销"功能 |
| **数据安全** | 避免误操作导致的数据丢失                     |
| **审计追踪** | 保留历史数据，便于审计和分析                 |
| **快速删除** | 仅需更新标记，操作瞬间完成                   |

#### 劣势

| 劣势             | 说明                                                     |
| ---------------- | -------------------------------------------------------- |
| **存储膨胀**     | 已删除数据持续占用存储空间，需要定期清理机制             |
| **查询复杂**     | 所有查询都需要添加 `isDeleted: false` 过滤条件           |
| **数据一致性**   | 需要确保所有查询点都正确应用过滤条件，容易遗漏           |
| **用户体验困惑** | 用户可能困惑为什么删除了还能看到（如果在某些地方没过滤） |

---

### 2.3 方案三：解除关联（保留资源）

**实现方式**: 删除剧本时，将关联资源的 `scriptId` 设为 `null`，使其成为"未分类"资源。

#### 优势

| 优势         | 说明                               |
| ------------ | ---------------------------------- |
| **资源复用** | 保留的资源可以在其他剧本中继续使用 |
| **避免误删** | 不会意外删除有价值的资源           |
| **灵活管理** | 用户可以在"未分类"中管理这些资源   |

#### 劣势

| 劣势         | 说明                                         |
| ------------ | -------------------------------------------- |
| **数据混乱** | "未分类"资源可能堆积，难以管理               |
| **用户困惑** | 用户删除剧本后，发现资源还在，可能不符合预期 |
| **存储占用** | 资源文件继续占用存储空间                     |

## 三、推荐方案

### 3.1 综合建议：级联删除 + 确认机制

**推荐采用方案一（级联删除）**，但需要增加以下保护措施：

#### 实现要点

1. **删除确认对话框**
   - 显示警告信息："删除剧本将同时删除关联的 X 个角色、Y 个场景、Z 个物品"
   - 需要用户二次确认

2. **实现级联删除逻辑**

```typescript
async deleteScript(scriptId: string, projectId: string): Promise<void> {
    // 1. 先获取关联资产统计（用于确认对话框显示）
    const assets = await this.getAssets(projectId);
    const relatedAssets = assets.filter(a => a.scriptId === scriptId);
    const stats = {
        characters: relatedAssets.filter(a => a.type === AssetType.CHARACTER).length,
        scenes: relatedAssets.filter(a => a.type === AssetType.SCENE).length,
        items: relatedAssets.filter(a => a.type === AssetType.ITEM).length
    };

    // 2. 删除剧本
    const filename = `scripts_${projectId}.json`;
    await this.lock(filename, async () => {
        let scripts = await this.getScripts(projectId);
        scripts = scripts.filter(s => s.id !== scriptId);
        await this.writeJson(filename, scripts);
    });

    // 3. 级联删除关联资产（使用现有的 deleteAsset 方法）
    for (const asset of relatedAssets) {
        await this.deleteAsset(asset.id, projectId);
    }

    return stats; // 返回删除统计
}
```

3. **UI 层增加确认对话框**

```typescript
const handleDeleteScript = async () => {
  if (!scriptToDelete || !projectId) return;

  try {
    // 获取关联资源统计
    const stats = await storageService.getScriptAssetStats(scriptToDelete.id, projectId);

    // 显示确认对话框（包含统计信息）
    const confirmed = await showDeleteConfirmDialog({
      title: '确认删除剧本',
      message: `删除剧本 "${scriptToDelete.name}" 将同时删除：`,
      details: [`${stats.characters} 个角色`, `${stats.scenes} 个场景`, `${stats.items} 个物品`],
      confirmText: '确认删除',
      cancelText: '取消',
      danger: true,
    });

    if (!confirmed) return;

    // 执行删除
    await storageService.deleteScript(scriptToDelete.id, projectId);
    showToast('剧本及关联资源删除成功', 'success');
  } catch (error: any) {
    showToast(`删除失败: ${error.message}`, 'error');
  }
};
```

### 3.2 备选方案：软删除（未来扩展）

如果未来需要支持"回收站"功能，可以：

1. 添加 `deletedAt` 字段到 Script 和 Asset 类型
2. 修改查询逻辑默认过滤 `deletedAt: null`
3. 提供"回收站"页面查看已删除项目
4. 提供"恢复"和"彻底删除"功能

## 五、测试要点

1. **正常删除**: 删除剧本后，关联角色/场景/物品不再显示
2. **空剧本删除**: 删除没有关联资源的剧本
3. **多剧本场景**: 确保只删除指定剧本的资源，不影响其他剧本
4. **错误处理**: 删除过程中断时的数据一致性
