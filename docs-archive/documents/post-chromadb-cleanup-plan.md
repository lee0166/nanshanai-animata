## 任务 2：创建功能缺失说明文档

### 文档位置

`docs/chromadb-removal-impact.md`

### 文档内容结构

#### 1. 移除的功能

| 功能           | 状态      | 说明                         |
| -------------- | --------- | ---------------------------- |
| 向量数据库存储 | ❌ 已移除 | 语义分块不再存储到向量数据库 |
| 智能上下文召回 | ❌ 已移除 | 基于相似度的上下文召回功能   |
| 长文本语义记忆 | ❌ 已移除 | 跨章节的语义关联记忆         |

#### 2. 保留的功能

| 功能           | 状态    | 说明                            |
| -------------- | ------- | ------------------------------- |
| 语义分块       | ✅ 保留 | SemanticChunker 仍然工作        |
| 上下文注入     | ✅ 保留 | ContextInjector 正常工作        |
| 全局上下文提取 | ✅ 保留 | GlobalContextExtractor 正常工作 |
| 基础剧本解析   | ✅ 保留 | 角色、场景、分镜提取正常        |

#### 3. 影响分析

**对短文本（< 1万字）**

- 影响：无
- 原因：短文本不需要向量记忆

**对中等文本（1-5万字）**

- 影响：轻微
- 原因：语义分块仍然有效，只是没有智能召回

**对长文本（> 5万字）**

- 影响：中等
- 原因：长文本的上下文连贯性可能下降

#### 4. 后续优化建议

**方案 A：DuckDB-Wasm（推荐）**

- 优点：纯浏览器、高性能、功能完整
- 工作量：约 2-3 天
- 优先级：高

**方案 B：后端托管**

- 优点：可扩展、企业级
- 工作量：约 1-2 周
- 优先级：中

**方案 C：简化上下文**

- 优点：实现简单
- 工作量：约 1 天
- 优先级：低
