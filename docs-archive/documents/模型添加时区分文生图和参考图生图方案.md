# 模型添加时区分文生图和参考图生图方案

## 问题背景

用户反馈：目前项目刷新后，在文生图模型选择中只能看到一个模型，但实际上除了 Seedream 4.0，其他模型都不支持参考图生图。项目显示上有错误。

## 根本原因

1. **DEFAULT_MODELS 配置问题** - 需要检查 `config/models.ts` 中各模型的 `capabilities.supportsReferenceImage` 配置是否正确
2. **自定义模型硬编码问题** - 自定义生图模型的 `supportsReferenceImage` 被硬编码为 `false`
3. **模型匹配逻辑问题** - 用户配置的模型ID可能与 DEFAULT_MODELS 中的ID不匹配

## 解决方案

### 阶段1：修正 DEFAULT_MODELS 配置

检查并修正 `config/models.ts` 中所有生图模型的 `supportsReferenceImage` 配置：

**应该支持参考图的模型：**

- Seedream 4.5 (volcengine) - supportsReferenceImage: true
- Seedream 4.0 (volcengine) - supportsReferenceImage: true
- Vidu Q2 (vidu) - supportsReferenceImage: true
- Vidu Q1 (vidu) - supportsReferenceImage: true
- Modelscope Z-Image (modelscope) - supportsReferenceImage: true

**不支持参考图的模型：**

- Seedream 3.0 T2I (volcengine) - supportsReferenceImage: false

### 阶段2：模型设置页面增加能力配置

在添加模型时，让用户明确选择模型能力：

#### 2.1 预设模型

预设模型从 DEFAULT_MODELS 复制 capabilities，保持现有逻辑。

#### 2.2 自定义模型增加配置选项

在自定义模型表单中增加：

- 是否支持参考图生图（Switch）
- 最大参考图数量（Number Input，仅在支持参考图时显示）

### 阶段3：模型列表显示优化

在模型设置页面的模型列表中：

- 显示模型类型标签（image/video/llm）
- 显示是否支持参考图的标识
- 让用户一目了然知道每个模型的能力

## 实施步骤

### 步骤1：检查并修正 DEFAULT_MODELS

1. 读取 `config/models.ts`
2. 检查每个生图模型的 `capabilities.supportsReferenceImage`
3. 修正错误的配置

### 步骤2：修改 Settings.tsx 自定义模型表单

1. 添加 `supportsReferenceImage` 状态
2. 添加 `maxReferenceImages` 状态
3. 在表单中增加 Switch 和 Number Input
4. 修改 `handleAddCustomModel` 使用这些值

### 步骤3：模型列表显示优化

1. 在模型列表中添加能力标识列
2. 使用 Chip 或图标显示是否支持参考图

## 预期效果

1. 文生图模式下只显示 `supportsReferenceImage: false` 的模型
2. 参考图生图模式下只显示 `supportsReferenceImage: true` 的模型
3. 用户添加自定义模型时可以明确配置能力
4. 模型列表清晰展示每个模型的能力
