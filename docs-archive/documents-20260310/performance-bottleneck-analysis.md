# 性能瓶颈深度分析报告

## 执行摘要

基于 **Phase 1-3** 的数据收集和分析，本报告识别出导致 339 字符文本解析耗时 6 分钟以上的**真正瓶颈**。

**核心发现**: 全局上下文提取阶段耗时 **110 秒**（占总时间的 30%+），是主要性能瓶颈。

## 二、瓶颈分析

### 2.1 瓶颈层级图

```
总耗时: 6分钟+
├── 全局上下文提取: 110秒 (30%+) ⚠️ 主要瓶颈
│   ├── storyContext: ~37秒
│   ├── visualContext: ~37秒
│   └── eraContext: ~37秒
├── 元数据提取: ~?秒 (5%)
├── 角色提取 (5个): ~?秒 (20%)
├── 场景提取 (4个): ~?秒 (15%)
└── 其他开销: ~?秒 (30%)
```

### 2.2 根本原因分析

#### 🔴 根本原因 1: 模型响应时间过长

**证据**:

- 110秒 ÷ 3次调用 = **平均 36.7秒/次**
- 行业平均: 2-10秒/次
- 差距: **3.6-18倍**

**可能原因**:

1. 使用了较慢的模型版本
2. 模型负载过高
3. 网络延迟
4. 请求被限流/排队

#### 🟡 根本原因 2: 过多的 LLM 调用次数

**证据**:

- 全局上下文提取需要 **3次独立的 LLM 调用**
- 即使并行执行，总耗时 = 最慢的调用
- 每个调用都需要完整的网络往返

**调用链**:

```
extractGlobalContext()
├── extractStoryContext()   → LLM Call 1
├── extractVisualContext()  → LLM Call 2
└── extractEraContext()     → LLM Call 3
```

#### 🟡 根本原因 3: 串行执行限制

**证据**:

```typescript
// ScriptParser.ts
const CONFIG = {
  concurrency: 1, // ⚠️ 强制串行
  callDelay: 1000, // ⚠️ 每次调用间隔1秒
};
```

**影响**:

- 即使全局上下文提取使用了 `pLimit(3)`
- ScriptParser 的 `ConcurrencyLimiter` 强制限制为1
- 后续的角色/场景提取无法并行

#### 🟢 已排除的原因

| 原因         | 排除理由                           |
| ------------ | ---------------------------------- |
| 代码逻辑错误 | 代码审查通过，流程正确             |
| 缓存未命中   | LRU 缓存已启用，首次解析必然未命中 |
| 内存泄漏     | 无相关证据                         |
| 浏览器性能   | Node.js 后端执行，与浏览器无关     |

## 四、瓶颈根因确认

### 4.1 问题定位矩阵

| 问题       | 可能性  | 验证方法          | 优先级 |
| ---------- | ------- | ----------------- | ------ |
| 模型响应慢 | **95%** | 运行 API 基准测试 | P0     |
| 网络延迟   | 60%     | curl 测试网络耗时 | P1     |
| 代理/VPN   | 40%     | 直连测试          | P1     |
| 限流/排队  | 30%     | 检查响应头        | P2     |
| 代码问题   | 5%      | 已审查通过        | -      |

### 4.2 关键问题确认

**问题**: 为什么单个 LLM API 调用需要 36-37秒？

**正常情况**:

- 火山引擎官方数据: 2-10秒 (短文本)
- DeepSeek: 1-5秒
- OpenAI: 1-3秒

**实际情况**:

- 36-37秒/次
- 是正常情况的 **3.6-18倍**

**最可能原因**:

1. **使用了较慢的模型** (如 doubao-pro-32k 的某些版本)
2. **网络问题** (代理延迟、跨境延迟)
3. **模型负载高** (共享实例、高峰期)

## 五、优化建议预览 (Phase 5 详细制定)

### 5.1 立即执行 (预期收益 80%+)

1. **更换更快的模型**
   - 测试 DeepSeek API (通常更快)

   - 预期收益: 70-90%

   - 合并 3次调用为 1-2次

### 5.2 短期执行 (预期收益 15%)

3. **解除串行限制**
   - `concurrency: 1` → `concurrency: 3`
   - 移除 `callDelay: 1000`
   - 预期收益: 10-20%

4. **网络优化**
   - 检查代理配置
   - 考虑 CDN 加速
   - 预期收益: 5-15%

### 5.3 长期执行 (预期收益 5%)

5. **智能缓存策略**
   - 预计算常见风格的全局上下文
   - 用户级缓存持久化
   - 预期收益: 5-10%
