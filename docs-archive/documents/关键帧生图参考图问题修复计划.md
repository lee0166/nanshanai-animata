# 关键帧生图参考图问题修复计划

## 问题描述

从日志分析发现，关键帧生图时参考图（角色图和场景图）没有正确传递给火山引擎生图模型。

### 日志证据

```json
[VolcengineProvider] Request body: {
  "model": "doubao-seedream-4-0-250828",
  "prompt": "参考角色图：001_LiuWei_YellowSkirt，参考场景图：002_GardenBackDoor；medium，柳唯站在花园大门外，电影级画质",
  "size": "2560x1440",
  "response_format": "url",
  "watermark": false
}
```

**问题：** 请求体中没有 `image` 字段，说明参考图没有被传递。

---

## 问题根因分析

### 1. 数据流追踪

```
ShotManager.handleGenerateImage (L553-L598)
    ↓ 准备 referenceImages (base64数组)
aiService.createKeyframeGenerationJob (L178-L205)
    ↓ 将 referenceImages 放入 job.params

    ↓ 从 job.params 取出 referenceImages

    ↓ 调用 loadBlobAsBase64 处理图片
    ↓ 调用 strategy.prepareRequest 构建请求体
```

### 2. 关键问题定位

#### 问题点1：VolcengineProvider 中 loadBlobAsBase64 的处理逻辑

**文件：** `services/ai/providers/VolcengineProvider.ts` L66-70

```typescript
// 1. Load Images
const loadedImages: string[] = [];
if (referenceImages && referenceImages.length > 0) {
  const blobs = await Promise.all(referenceImages.map(img => this.loadBlobAsBase64(img)));
  loadedImages.push(...blobs.filter((b): b is string => !!b));
}
```

**问题：** `loadBlobAsBase64` 在 `BaseProvider.ts` L23-42 中：

```typescript
protected async loadBlobAsBase64(urlOrPath: string): Promise<string | null> {
     if (urlOrPath.startsWith('data:')) return urlOrPath;  // ✅ 已经是base64，直接返回

     try {
         let blob: Blob;
         if (urlOrPath.startsWith('http')) {
            const res = await this.makeRequest(urlOrPath);
            blob = await res.blob();
        } else {
             const storageUrl = await storageService.getAssetUrl(urlOrPath);
             if (!storageUrl) return null;
             const res = await fetch(storageUrl);
             blob = await res.blob();
         }
         return await this.blobToBase64DataUri(blob);
     } catch (e) {
         console.error("Failed to load blob as base64", e);
         return null;
     }
}
```

**ShotManager 中转换的 base64 格式：**

```typescript
return `data:image/${ext};base64,${base64}`;
```

**问题分析：**

- ShotManager 生成的格式是：`data:image/png;base64,abc123...`

- `loadBlobAsBase64` 检查 `startsWith('data:')` 应该返回 `true`

- 所以应该直接返回，不会返回 `null`

**需要验证：** 是否 `referenceImages` 数组在传递过程中变成了空数组？

#### 问题点2：executeKeyframeGenerationJob 中参数解构

**文件：** `services/queue.ts` L597-L600

```typescript

  scriptId,
```

**问题：** 需要确认 `job.params.referenceImages` 是否存在且非空。

#### 问题点3：日志缺失

在 `VolcengineProvider.generateImage` 中没有打印 `referenceImages` 的日志，无法直接确认传入的参数。

### 阶段2：根据日志定位具体问题

根据阶段1的日志输出，可能出现的情况：

#### 情况A：ShotManager 中 referenceImages 为空

- **原因：** 角色图或场景图路径获取失败

- **修复：** 检查 `characterAsset` 和 `sceneAsset` 的获取逻辑

#### 情况B：job.params 中 referenceImages 为空

- **原因：** `createKeyframeGenerationJob` 没有正确传递参数

- **修复：** 检查 `aiService.createKeyframeGenerationJob`

#### 情况C：VolcengineProvider 中 referenceImages 为空

- **原因：** `aiService.generateImage` 调用时参数丢失

- **修复：** 检查 `aiService.generateImage` 的参数传递

#### 情况D：loadedImages 为空但 referenceImages 有数据

- **原因：** `loadBlobAsBase64` 返回 null

- **修复：** 检查 `loadBlobAsBase64` 的处理逻辑
