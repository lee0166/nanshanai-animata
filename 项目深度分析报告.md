## 一、项目概览

### 1.1 项目定位

**Kmeng AI Animata** 是一个基于 **Local-First（本地优先）** 架构的 AI 影视资产生成平台，专注于剧本解析、角色/场景/分镜生成、视频制作的全流程影视创作工具。

### 1.2 核心价值主张

- ✅ **完全本地化**: 所有数据存储在用户本地，无云端依赖
- ✅ **隐私安全**: 数据不上传，保障用户隐私
- ✅ **离线可用**: 无网络也能正常工作
- ✅ **多厂商适配**: 支持火山引擎、Vidu、Modelscope、阿里云、OpenAI等多个AI厂商

## 二、核心架构设计

### 2.1 Local-First 架构理念

项目采用 **Local-First** 作为核心设计理念，具体体现在：

#### 2.1.1 双模式存储系统

项目支持两种本地存储模式：

1. **File System Access API 模式**
   - 用户主动选择文件夹作为工作区
   - 文件直接存储在用户指定的本地文件夹
   - 支持文件的直接编辑和查看
   - 适用场景：需要直接访问文件的用户

2. **Origin Private File System (OPFS) 模式**
   - 浏览器私有文件系统
   - 数据隔离，隐私性更强
   - 适用场景：追求隐私的用户

**核心实现文件**: `services/storage.ts:1-xxxx`

#### 2.1.2 多层缓存机制

```
L1 内存缓存 → L2 TTL缓存 → 持久化存储
```

- **L1 缓存**: 内存缓存，快速访问
- **L2 缓存**: TTL 缓存，带过期时间
- **持久化**: 本地文件系统存储

## 三、核心模块深度分析

### 3.1 剧本解析模块 (Script Parser)

**文件位置**: `services/scriptParser.ts`  
**文件大小**: 50KB+  
**重要程度**: ⭐⭐⭐⭐⭐ (核心核心模块)

#### 3.1.1 解析流水线

项目实现了完整的 **4阶段线性解析流水线**：

```
文本预处理
    ↓
元数据提取 (metadata)
    ↓
角色提取 (characters)
    ↓
场景提取 (scenes)
    ↓
分镜生成 (shots)
```

**但实际上，项目实际实现了更先进的 **5阶段闭环解析\*\*：

```
文本预处理
    ↓
全局上下文提取 (GlobalContextExtractor)
    ↓
元数据 + 角色提取
    ↓
场景提取
    ↓
分镜生成
    ↓
质量分析 (QualityAnalyzer)
```

#### 3.1.2 关键子模块

**1. GlobalContextExtractor（全局上下文提取器）**

**文件位置**: `services/parsing/GlobalContextExtractor.ts:1-944`

**功能**: 在元数据阶段提取全局上下文信息，为后续所有解析阶段提供统一的背景知识

**提取内容**:

- ✅ **故事核心**: 梗概、冲突、主题、结构
- ✅ **视觉风格**: 美术指导、色彩方案、摄影风格、光影
- ✅ **时代背景**: 年代、地点、季节
- ✅ **情绪曲线**: 情绪起伏图谱
- ✅ **一致性规则**: 时代限制、风格限制、禁止元素

**关键方法**:

```typescript
extract(); // 主入口：单次LLM调用提取所有上下文
extractUnifiedContext(); // V2优化：单次调用替代3次调用
extractStoryContext(); // 提取故事核心
extractVisualContext(); // 提取视觉风格
extractEraContext(); // 提取时代背景
extractEmotionalArc(); // 提取情绪曲线
generateConsistencyRules(); // 生成一致性规则
```

**2. ContextInjector（上下文注入器）**

**文件位置**: `services/parsing/ContextInjector.ts:1-525`

**功能**: 将全局上下文注入到各解析阶段的Prompt中

**关键方法**:

```typescript

```

**3. QualityAnalyzer（质量分析器）**

**文件位置**: `services/parsing/QualityAnalyzer.ts:1-866`

**功能**: 多维度质量分析服务

**5维度质量分析**:

1. **COMPLETENESS (完整性)**: 字段完整性、分镜覆盖度
2. **ACCURACY (准确性)**: 角色名准确性、分镜参数准确性
3. **CONSISTENCY (一致性)**: 角色一致性、场景一致性
4. **USABILITY (可用性)**: 角色/场景/分镜可用性
5. **DRAMATIC (戏剧性)**: 短剧规则校验
