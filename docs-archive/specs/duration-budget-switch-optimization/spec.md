# 时长预算功能开关联动优化 Spec

## Why

当前设置页面中的功能开关存在严重的**用户体验断层**和**功能失效陷阱**：

1. **隐性依赖导致功能失效**：`启用时长预算`必须配合`启用生产级Prompt`才能生效，但UI中两个开关独立显示。用户可能只开启前者而忘记后者，导致时长预算计算了但并未应用到分镜生成，造成**静默失败**。

2. **状态不透明**：用户无法直观看到哪些开关已真正生效。开启`启用时长预算`后，界面上没有任何反馈表明它正在工作，也没有任何提示表明它因为缺少依赖而无效。

3. **认知负担过重**：5个功能开关平铺展示，没有分组、没有优先级、没有推荐配置，用户不知道该如何组合使用。

4. **场景适配缺失**：不同字数的小说（短文本/中篇/长篇）和不同平台（抖音/快手/B站/精品）需要不同的配置策略，但系统没有提供智能推荐。

5. **未实现功能造成困惑**：`启用动态时长`和`自动调整不符合预算的分镜`两个开关在UI中可交互，但功能未实现，用户开启后没有反应，造成**信任危机**。

---

## What Changes

### 核心设计原则

- **显式优于隐式**：所有依赖关系必须可视化，用户必须清楚知道开关之间的关联
- **预防优于补救**：在用户做出可能导致功能失效的操作前，提前警告并提供解决方案
- **智能优于手动**：根据场景（字数、平台）智能推荐最优配置，降低决策成本
- **渐进式披露**：高级功能默认隐藏，基础功能一键配置

### 具体变更

- **BREAKING**: 重构功能开关架构，从"独立开关"改为"配置组"模式
- **BREAKING**: 彻底移除未实现的功能开关（useDynamicDuration、qcAutoAdjust），避免用户困惑
- **ADDED**: 引入"配置组"概念，将相关开关组织为逻辑单元
- **ADDED**: 实现开关依赖关系图，可视化显示依赖链条
- **ADDED**: 添加配置状态面板，实时显示当前配置的有效性
- **ADDED**: 实现智能配置引擎，基于字数和平台推荐最优配置
- **ADDED**: 提供平台模板一键配置（抖音/快手/B站/精品）
- **ADDED**: 添加场景化引导（上传小说后自动推荐配置）

---

## Impact

- **用户体验提升**：消除功能失效陷阱，降低认知负担，提升配置效率
- **功能利用率提升**：通过智能推荐和场景化引导，让更多用户正确使用时长预算功能
- **开发维护简化**：移除未实现功能，减少技术债务
- **商业价值提升**：帮助用户生成更符合平台规范的内容，提升内容质量和传播效果

## ADDED Requirements

### Requirement: 配置组架构（Configuration Group Architecture）

The system SHALL organize related switches into logical configuration groups.

#### Rationale

将独立开关组织为配置组，可以：

1. 清晰表达功能之间的逻辑关系
2. 支持批量配置和一键切换
3. 降低用户的认知负担
4. 为智能推荐提供结构化基础

#### Scenario: 配置组显示

- **GIVEN** 用户进入时长预算配置页面
- **THEN** 系统显示三个配置组：
  - **基础配置组**（平台、节奏）- 必须配置
  - **核心功能组**（时长预算、生产级Prompt）- 联动配置
  - **高级功能组**（分镜质检）- 可选配置

#### Scenario: 配置组联动

- **GIVEN** 用户开启"核心功能组"中的"启用时长预算"
- **WHEN** "启用生产级Prompt"为关闭状态
- **THEN** 系统自动开启"启用生产级Prompt"
- **AND** 显示提示："已自动开启关联功能"

### Requirement: 依赖关系可视化（Dependency Visualization）

The system SHALL visualize the dependency relationships between switches.

#### Rationale

可视化依赖关系可以：

1. 让用户清楚了解功能之间的关联
2. 预防因关闭依赖项导致的功能失效
3. 提供直观的配置指导

#### Scenario: 依赖关系图显示

- **GIVEN** 用户查看"核心功能组"
- **THEN** 系统显示依赖关系图：
  ```
  启用时长预算 ──依赖──> 启用生产级Prompt
       │
       └──依赖──> 目标平台
       │
       └──依赖──> 节奏选择
  ```

#### Scenario: 依赖项关闭警告

- **GIVEN** "启用时长预算"和"启用生产级Prompt"均为开启状态
- **WHEN** 用户尝试关闭"启用生产级Prompt"
- **THEN** 系统显示警告对话框：
  - 标题："关闭此功能将影响其他功能"
  - 内容："关闭'启用生产级Prompt'将导致'启用时长预算'失效，因为时长预算约束需要通过生产级Prompt注入到AI生成过程中。"
  - 选项：
    - "同时关闭时长预算（推荐）"
    - "仅关闭生产级Prompt（时长预算将失效）"
    - "取消"

### Requirement: 智能配置引擎（Smart Configuration Engine）

The system SHALL recommend optimal configuration based on novel length and target platform.

#### Rationale

智能推荐可以：

1. 降低用户的决策成本
2. 确保配置的最优性
3. 提升功能利用率

#### Scenario: 短文本推荐（<3000字）

- **GIVEN** 用户上传了2000字的小说
- **WHEN** 系统检测到字数
- **THEN** 智能配置引擎推荐：

  ```
  📊 智能配置推荐

  基于您的文本（2000字），系统推荐以下配置：

  推荐配置：轻量级模式
  - 目标平台：抖音
  - 节奏：快
  - 时长预算：❌ 关闭（推荐）
  - 生产级Prompt：❌ 关闭（推荐）

  推荐理由：
  短文本（<3000字）生成视频时长通常在1-2分钟，
  不需要复杂的时长预算规划。使用标准Prompt即可
  获得良好的生成效果。

  [应用推荐配置] [自定义配置]
  ```

#### Scenario: 中篇文本推荐（3000-8000字）

- **GIVEN** 用户上传了7000字的小说
- **WHEN** 系统检测到字数
- **THEN** 智能配置引擎推荐：

  ```
  📊 智能配置推荐

  基于您的文本（7000字），系统推荐以下配置：

  推荐配置：专业模式
  - 目标平台：抖音
  - 节奏：快
  - 时长预算：✅ 开启（推荐）
  - 生产级Prompt：✅ 开启（推荐）
  - 分镜质检：⚠️ 可选

  推荐理由：
  中篇文本（3000-8000字）生成视频时长通常在3-5分钟，
  需要时长预算规划来确保内容符合平台规范。
  预计生成：25-35个分镜，总时长3.5-5分钟。

  [应用推荐配置] [自定义配置]
  ```

#### Scenario: 长篇文本推荐（>8000字）

- **GIVEN** 用户上传了12000字的小说
- **WHEN** 系统检测到字数
- **THEN** 智能配置引擎推荐：

  ```
  📊 智能配置推荐

  基于您的文本（12000字），系统推荐以下配置：

  推荐配置：高级模式
  - 目标平台：B站

  - 时长预算：✅ 开启（推荐）

  建议选择B站平台（支持更长视频）或精品模式。
  预计生成：50-70个分镜，总时长8-12分钟。

  [应用推荐配置] [自定义配置]
  ```
