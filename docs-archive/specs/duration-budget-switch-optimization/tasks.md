# 时长预算功能开关联动优化 - 任务列表

> **change-id**: duration-budget-switch-optimization  
> **目标**: 优化功能开关的联动逻辑和用户体验，消除功能失效陷阱  
> **原则**: 显式优于隐式、预防优于补救、智能优于手动、渐进式披露

---

## Phase 1: P0 - 解决核心痛点（必须）

### Task 1.1: 移除未实现的功能开关

**描述**: 从UI中彻底移除useDynamicDuration和qcAutoAdjust两个未实现的功能开关，避免用户困惑和信任危机
**验收标准**:

- [ ] 从Settings.tsx中移除"启用动态时长"开关
- [ ] 从Settings.tsx中移除"自动调整不符合预算的分镜"开关
- [ ] 确保配置项仍保留在代码中（未来实现用）
- [ ] 更新相关类型定义和接口
- [ ] 更新CHANGELOG.md记录此变更
      **依赖**: 无
      **预计工时**: 1h
      **优先级**: P0

### Task 1.2: 实现配置组架构

**描述**: 重构设置页面UI，将功能开关组织为三个逻辑配置组：基础配置组、核心功能组、高级功能组
**验收标准**:

- [ ] 创建基础配置组组件（平台、节奏）
- [ ] 创建核心功能组组件（时长预算、生产级Prompt）
- [ ] 创建高级功能组组件（分镜质检）
- [ ] 实现配置组的视觉分组和标签
- [ ] 确保配置持久化正常工作
      **依赖**: Task 1.1
      **预计工时**: 2h
      **优先级**: P0

### Task 1.3: 实现开关依赖关系检测

**描述**: 实现时长预算与生产级Prompt之间的依赖关系检测逻辑，当依赖未满足时给出明确提示
**验收标准**:

- [ ] 创建开关依赖关系映射表（durationBudget -> productionPrompt）
- [ ] 实现依赖检测函数checkDependencyStatus()
- [ ] 当用户开启durationBudget但productionPrompt关闭时，显示警告提示
- [ ] 提供"同时开启关联功能"快捷操作按钮
- [ ] 依赖状态变化时实时更新UI
      **依赖**: Task 1.2
      **预计工时**: 2h
      **优先级**: P0

### Task 1.4: 实现配置状态面板

**描述**: 添加实时配置状态面板，显示当前配置的有效性、依赖状态和建议
**验收标准**:

- [ ] 创建ConfigurationStatusPanel组件
- [ ] 实现状态计算逻辑（有效/无效/依赖未满足）
- [ ] 显示基础配置状态（平台、节奏）
- [ ] 显示核心功能状态（时长预算、生产级Prompt及依赖关系）
- [ ] 显示高级功能状态（分镜质检）
- [ ] 当配置无效时，显示具体原因和解决方案
- [ ] 添加配置优化建议
      **依赖**: Task 1.3
      **预计工时**: 2.5h
      **优先级**: P0

### Task 1.5: 实现关闭依赖项的确认对话框

**描述**: 当用户尝试关闭被依赖的功能开关时，显示确认对话框，说明影响并提供解决方案
**验收标准**:

- [ ] 监听productionPrompt开关的关闭事件
- [ ] 当durationBudget开启时，拦截关闭操作
- [ ] 显示确认对话框，说明关闭productionPrompt将导致durationBudget失效
- [ ] 提供三个选项：
  - "同时关闭时长预算（推荐）"
  - "仅关闭生产级Prompt（时长预算将失效）"
  - "取消"
- [ ] 用户选择后正确执行对应操作
      **依赖**: Task 1.3
      **预计工时**: 1.5h
      **优先级**: P0

## Phase 2: P1 - 提升用户体验（重要）

### Task 2.1: 实现平台模板快速选择

**描述**: 添加平台模板按钮（抖音/快手/B站/精品），一键应用经过验证的配置组合
**验收标准**:

- [ ] 创建PlatformTemplateSelector组件
- [ ] 实现抖音短剧模板（竖屏、快节奏、时长预算开启）
- [ ] 实现快手短剧模板（竖屏、中节奏、时长预算开启）
- [ ] 实现B站视频模板（横屏、中节奏、时长预算开启、质检开启）
- [ ] 实现精品短剧模板（横屏、慢节奏、时长预算开启、质检开启）
- [ ] 点击模板后自动配置所有相关设置
- [ ] 显示模板应用成功提示和配置详情
- [ ] 支持自定义修改（应用模板后可手动调整）
      **依赖**: Task 1.2
      **预计工时**: 2.5h
      **优先级**: P1

### Task 2.2: 实现智能配置引擎

**描述**: 基于小说字数和目标平台，智能推荐最优配置
**验收标准**:

- [ ] 创建SmartConfigEngine服务
- [ ] 实现字数检测逻辑（<3000字/3000-8000字/>8000字）
- [ ] 实现短文本推荐逻辑（轻量级模式：关闭时长预算）
- [ ] 实现中篇文本推荐逻辑（专业模式：开启时长预算）
- [ ] 实现长篇文本推荐逻辑（高级模式：开启时长预算+分块解析）
- [ ] 实现平台适配逻辑（抖音/快手/B站/精品的不同推荐）
- [ ] 生成推荐理由和预计效果（时长、分镜数）
- [ ] 提供"应用推荐配置"按钮
      **依赖**: Task 2.1
      **预计工时**: 3h
      **优先级**: P1

### Task 2.3: 实现依赖关系可视化

**描述**: 在核心功能组中显示依赖关系图，让用户清楚了解功能之间的关联
**验收标准**:

- [ ] 创建DependencyGraph组件
- [ ] 使用图形化方式展示依赖关系（节点+连线）
- [ ] 时长预算节点显示为"依赖方"
- [ ] 生产级Prompt节点显示为"被依赖方"
- [ ] 使用不同颜色表示依赖状态（绿色=满足、红色=未满足）
- [ ] 鼠标悬停显示依赖说明
- [ ] 依赖状态变化时图形实时更新
      **依赖**: Task 1.3
      **预计工时**: 2h
      **优先级**: P1

- [ ] 提供"稍后提醒"选项（下次上传时再次提示）

## 风险评估

### 高风险

- **风险**: 移除未实现功能开关可能影响现有用户配置
- **缓解**: 保留配置项在代码中，仅从UI移除，确保向后兼容

### 中风险

- **风险**: 开关依赖关系检测逻辑复杂，可能引入bug
- **缓解**: 编写详细的单元测试，覆盖所有依赖场景

### 低风险

- **风险**: UI视觉设计改动可能影响用户习惯
- **缓解**: 保持核心交互不变，仅优化视觉呈现
