## 1. 功能模块概述

### 1.1 功能定位

"确认解析配置"弹窗是小说文本导入流程中的关键前置环节，位于**大模型解析开始之前**，其核心目标是：

- 避免无意识的高成本解析操作
- 提供智能化配置推荐
- 让用户在解析前确认关键参数
- 估算解析成本与时长

### 1.2 涉及的核心文件

| 文件路径 | 功能 |
|---------|------|
| `components/ParseConfigConfirmModal.tsx` | 配置确认弹窗主组件 |
| `components/DurationBudget/SmartConfigEngine.tsx` | 智能配置引擎 |
| `services/parsing/BudgetPlanner.ts` | 时长预算计算引擎 |
| `services/textCleaner.ts` | 文本清洗与字数统计 |
| `views/ScriptManager.tsx` | 剧本管理页（调用方） |

### 2.1 字数统计与Tokens预测功能

#### 2.1.1 整体实现流程

```
用户上传文本 
  ↓
TextCleaner.process() 清洗
  ↓
计算清洗后文本长度
  ↓
估算Tokens数量
  ↓
在弹窗中显示
```

#### 2.1.2 字数统计实现

**计数规则**（`ScriptManager.tsx` 第285行、1157行）：

```typescript
// 在文件上传时
const wordCount = cleanResult.cleanedText.length;

// 在实时编辑时
const newWordCount = newContent.length;
```

**特点**：
- **计数方式**：简单的字符长度统计（`String.length`）
- **包含内容**：所有可见字符（中文字符、英文字符、标点符号、换行符等）
- **前置处理**：经过 `TextCleaner` 清洗后的文本

**TextCleaner清洗流程**（`textCleaner.ts` 第29-58行）：

1. 统一换行符：`\r\n`、`\r` → `\n`
2. 去除控制字符（保留换行、制表符）
3. 合并多余空行（3个及以上换行合并为2个）
4. 去除行首行尾空白
5. 统一空格（多个空格合并为一个）
6. 去除全文首尾空白

#### 2.1.3 Tokens预测算法实现

**实现代码**（`ParseConfigConfirmModal.tsx` 第117-120行）：

```typescript
const estimateTokenCount = (wordCount: number) => {
  // 中文字符约 0.5-1 Token/字
  return Math.ceil(wordCount * 0.8);
};
```

**算法特点**：
- **简化模型**：使用线性估算 `字数 × 0.8`
- **Token/字比例**：0.8（中文字符通常0.5-1 Token/字）
- **向上取整**：使用 `Math.ceil()` 确保预估充足
- **局限性**：不区分中/英文字符，不使用真实分词器

#### 2.1.4 数据流向

```
ScriptManager (调用方)
  ↓
传递 wordCount 给 ParseConfigConfirmModal
  ↓
ParseConfigConfirmModal 内部计算 estimatedTokens
  ↓
在UI上显示："{wordCount} 字 · ~{estimatedTokens} Tokens"
```

### 2.4 时长预算配置功能

#### 2.4.1 计算模型架构

时长预算配置的核心计算逻辑位于 `services/parsing/BudgetPlanner.ts`。

#### 2.4.2 影响因素

**主要影响因素**：
1. **文本字数**：基础输入
2. **目标平台**：决定字数/分钟基准
3. **节奏类型**：快/中/慢，影响字数/分钟
4. **场景重要性**：开场/发展/高潮/结尾，权重不同

#### 2.4.3 核心计算公式

**基础公式**（`BudgetPlanner.ts` 第424行）：

```typescript
const baseDuration = (totalWordCount / wordsPerMinute) * 60;
```

**字数/分钟配置**（`BudgetPlanner.ts` 第147-252行）：

| 平台 | 节奏 | 最小 | 最大 | 推荐 |
|------|------|------|------|------|
| 抖音 | 快 | 250 | 300 | 280 |
| 抖音 | 中 | 200 | 250 | 225 |
| 抖音 | 慢 | 150 | 200 | 175 |
| B站 | 快 | 220 | 280 | 250 |
| B站 | 中 | 180 | 250 | 200 |
| B站 | 慢 | 120 | 180 | 150 |

**场景时长分配公式**（`BudgetPlanner.ts` 第436-444行）：

```typescript
const weightRatio = scene.weight / totalWeight;
const wordRatio = scene.wordCount / totalWordCount;
const combinedRatio = weightRatio * 0.6 + wordRatio * 0.4;
const allocatedDuration = targetTotalDuration * combinedRatio;
```

**权重分配**：
- 权重占比：60%
- 字数占比：40%

**场景重要性权重**（`BudgetPlanner.ts` 第31-36行）：

```typescript
const SCENE_IMPORTANCE_WEIGHTS = {
  opening: 0.8,      // 开场
  development: 0.6,  // 发展
  climax: 1.0,       // 高潮（权重最高）
  ending: 0.7        // 结尾
};
```

#### 2.4.4 推荐分镜数计算

**计算公式**（`BudgetPlanner.ts` 第484-485行）：

```typescript
const averageShotDuration = targetTotalDuration / shots.length;
const recommendedShotCount = Math.ceil(targetTotalDuration / 4); // 假设平均每镜4秒
```

#### 2.4.5 预算验证机制

**验证项**（`BudgetPlanner.ts` 第511-559行）：

1. **7000字目标范围检查**：210-300秒（3.5-5分钟）
2. **平均每镜时长检查**：1.5-8秒
3. **场景时长分配总和验证**
4. **高潮部分占比检查**：15%-40%

## 3. 功能模块间协同工作机制

### 3.1 数据交互时序

```
ScriptManager
  │
  ├─ 用户上传/编辑文本
  │   ↓
  ├─ TextCleaner.process() 清洗文本
  │   ↓
  ├─ 计算 wordCount = cleanedText.length
  │   ↓
  ├─ 用户点击"一键解析"
  │   ↓
  ├─ setShowParseConfirm(true) 打开弹窗
  │   ↓
  └─ 传递参数给 ParseConfigConfirmModal
      ├─ scriptTitle
      ├─ wordCount
      ├─ modelName
      ├─ parseMode
      ├─ durationBudgetConfig (当前配置)
      └─ onConfigChange (配置变更回调)
      
ParseConfigConfirmModal
  │
  ├─ 接收 wordCount
  │   ↓
  ├─ 计算 estimatedTokens = wordCount × 0.8
  │   ↓
  ├─ generateSmartRecommendation() 生成推荐
  │   ├─ 根据 wordCount 判断模式
  │   ├─ 生成 recommendedConfig
  │   └─ 生成 estimatedDuration / recommendedShots
  │
  ├─ 用户选择平台模板 / 应用智能推荐
  │   ↓
  ├─ 调用 onConfigChange 更新配置
  │   ↓
  └─ 用户点击"开始解析"
      ↓
ScriptManager
  │
  ├─ setShowParseConfirm(false) 关闭弹窗
  │   ↓
  ├─ 将 durationBudgetConfig 传入 parserConfig
  │   ↓
  └─ 调用 createScriptParser() 开始解析
```

### 3.2 模块依赖关系

```
ScriptManager (调用方)
  │
  ├─ 依赖 TextCleaner (文本清洗)
  │
  └─ 调用 ParseConfigConfirmModal (弹窗组件)
      │
      ├─ 内部集成智能推荐算法
      │
      └─ 传递配置回 ScriptManager
          │
          └─ ScriptManager 传递给 BudgetPlanner (时长计算)
              │
              └─ 在解析过程中使用
```

### 3.4 开关联动逻辑

**智能开关联动**（`ParseConfigConfirmModal.tsx` 第256-277行）：

```typescript
const toggleSwitch = (key: string, value: boolean) => {
  if (key === 'useDurationBudget' && value) {
    // 开启时长预算时，自动开启生产级Prompt
    setConfig({
      ...config,
      useDurationBudget: true,
      useProductionPrompt: true,
    });
  } else if (key === 'useProductionPrompt' && !value && config.useDurationBudget) {
    // 关闭生产级Prompt时，如果时长预算已开启，同时关闭时长预算
    setConfig({
      ...config,
      useProductionPrompt: false,
      useDurationBudget: false,
    });
  } else {
    setConfig({
      ...config,
      [key]: value,
    });
  }
};
```

**联动规则**：
- 开启 `useDurationBudget` → 自动开启 `useProductionPrompt`
- 关闭 `useProductionPrompt` 且 `useDurationBudget` 已开启 → 同时关闭 `useDurationBudget`

## 6. 异常处理机制

**预算计算异常**：

- 对不支持的平台/节奏有错误抛出

### 6.2 潜在异常场景

| 场景 | 风险 | 当前处理 |
|------|------|---------|
| 极长文本（&gt;10万字） | Tokens估算偏差大 | 仅显示警告，无特殊处理 |
| 混合语言文本 | Token估算不准确 | 统一按0.8计算 |
| 用户手动修改所有开关 | 配置可能不合理 | 无自动校验 |
| 网络中断 | 配置丢失 | 配置仅在内存中 |

### 6.3 建议的异常处理增强

1. **添加配置合理性验证**：在用户确认前检查配置是否合理
2. **添加配置持久化**：将用户配置保存到localStorage
3. **添加Undo/Redo**：支持配置修改的撤销和重做
4. **添加更细致的警告**：针对不同异常场景提供具体建议
