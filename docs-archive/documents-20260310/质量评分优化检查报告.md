## 根本原因

**QualityReportCard 组件未被使用**

分析结果显示：

1. **代码已实现**：
   - `services/parsing/QualityAnalyzer.ts` - 增强了问题信息（target, context, targetId, targetType）
   - `components/ScriptParser/QualityReportCard.tsx` - 新的UI组件（IssueCard、按维度分组、Tab切换）

2. **实际使用情况**：
   - `views/ScriptManager.tsx` 第1135-1312行 - 质量评估Tab使用的是**内联代码**，直接遍历 `qualityReport.violations`
   - 没有导入和使用 `QualityReportCard` 组件
   - 显示的是简化版问题列表，只显示 `v.message`，没有展示 `target` 和 `context`

## 解决方案

### 方案：将 QualityReportCard 组件集成到 ScriptManager

**修改文件**：`views/ScriptManager.tsx`

**需要做的修改**：

1. 导入 QualityReportCard 组件
2. 在质量评估 Tab 中替换现有代码为 QualityReportCard 组件
3. 传递必要的 props：
   - `report`: DetailedQualityReport 格式的报告数据
   - `onFixIssue`: 修复问题的回调函数
   - `onNavigateToTarget`: 跳转回调函数
   - `t`: 翻译函数

**详细修改点**：

```typescript
// 1. 导入组件
import QualityReportCard from '../components/ScriptParser/QualityReportCard';
import { DetailedQualityReport } from '../services/parsing/QualityAnalyzer';

// 2. 在质量评估 Tab 中替换为：
<QualityReportCard
  report={qualityReport as DetailedQualityReport}
  onFixIssue={(issue) => console.log('Fix:', issue)}
  onNavigateToTarget={(type, id) => console.log('Navigate:', type, id)}
  t={t}
/>
```

### 需要注意的问题

1. **数据类型转换**：
   - 当前存储的 `qualityReport` 是 `QualityReport` 类型
   - `QualityReportCard` 需要 `DetailedQualityReport` 类型
   - 需要确保数据完整性或做类型适配

2. **回调函数实现**：
   - `onFixIssue` - 需要实现具体的修复逻辑
   - `onNavigateToTarget` - 需要实现页面跳转逻辑
