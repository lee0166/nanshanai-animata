# 场景和物品显示问题根因分析

## 日志分析结果

### 场景管理

```
[AssetList] Before filter: (2) [
  {id: 'scene_1771252433822_op3ror8hi', name: '花园', scriptId: undefined},
  {id: 'scene_1771252303849_0k6m9xvp7', name: '阁楼', scriptId: undefined}
]
[AssetList] After filter: []
```

**问题根因：**

- 场景资产存在（2个：花园、阁楼）
- 但 `scriptId` 是 `undefined`
- 用户选择了"初夏"剧本进行筛选
- 筛选条件：`asset.scriptId === 'script_1771252153864_5f83y0gkg'`
- 结果：没有匹配项，显示为空

### 物品管理

```
[AssetList] Before filter: []
```

**问题根因：**

- 项目中没有物品资产
- 不是筛选问题，是真的没有数据

## 问题总结

| 页面     | 问题       | 原因                                           |
| -------- | ---------- | ---------------------------------------------- |
| 场景管理 | 不显示场景 | 场景的 `scriptId` 为 `undefined`，筛选后无匹配 |
| 物品管理 | 不显示物品 | 项目中没有物品资产                             |

## 为什么场景的 scriptId 是 undefined？

可能原因：

1. 创建场景时用户没有选择剧本（可选字段）
2. 或者选择后没有正确保存

## 修复方案

### 方案1：优化筛选逻辑（推荐）

当前筛选逻辑：

- "全部剧本"：显示所有资产
- "未分类"：显示 `scriptId` 为空的资产
- 具体剧本：显示匹配该剧本的资产

**问题**：用户可能期望看到所有资产，包括无剧本的

**优化**：

- 保持现有逻辑不变
- 但需要在UI上明确提示："当前筛选剧本：XXX，无匹配资产"
- 或者添加"显示全部"选项

### 方案2：重新创建场景时选择剧本

- 删除现有场景
- 重新创建并选择剧本
- 但老数据无法修复

### 方案3：给无剧本资产自动分配

- 如果资产没有 `scriptId`，自动分配给默认剧本
- 但可能不符合用户意图

## 推荐方案

**方案1：优化筛选逻辑 + UI提示**

1. 当筛选结果为0时，显示提示：
   - "当前剧本下无场景，请创建新场景或选择其他剧本"
   - "或选择"未分类"查看无剧本的场景"

2. 添加快速切换：
   - 一键切换到"全部剧本"
   - 一键切换到"未分类"

这样用户就能理解为什么看不到资产。
