# 行业小说转影视AI解析流程调研报告

## 调研目标

了解当前行业中领先的小说转影视/短剧AI应用在文本清洗、解析处理方面的最佳实践，为本项目提供优化建议。

---

## 一、行业领先产品分析

### 1.1 国内主流AI短剧/影视生成平台

#### **白日梦AI (Daydream AI)**

- **文本处理流程**：
  1. 智能识别角色对话、场景切换
  2. 自动生成分镜脚本：「咖啡馆特写→男主低头看手机→女主推门带风铃响」
  3. 支持小说片段输入，自动识别关键画面

- **技术特点**：
  - 自动分镜提取，识别镜头语言描述
  - 情绪识别与标注
  - 场景切换自动检测

#### **模力漫AI**

- **智能分析系统**：
  1. **动作指令识别**："女主攥紧拳头后退"、"男主摔文件"
  2. **情绪标注**：自动标注角色情绪状态
  3. **场景分类**：室内/室外、白天/夜晚自动识别

- **分镜策略**：
  - 精准抓取3类必拆关键信息
  - 传统逐字拆分→智能关键画面提取

#### **Seko 2.0 / 魔因漫创**

- **剧本解析能力**：
  - 粘贴剧本文字，AI智能拆解
  - 自动解析为：场景、分镜、对白结构化元素
  - 精准识别：角色、场景、情绪、镜头语言

- **导演视角编辑器**：
  - 主角与场景组合
  - 生成连贯分镜图
  - 故事板呈现

#### **SkyReels (昆仑万维)**

- **角色一致性方案**：
  1. 角色基础设定提取
  2. 分镜脚本生成
  3. 视觉转化（三视图生成）

- **技术亮点**：
  - 角色三视图（正面、侧面、背面）
  - 图生图功能维持角色一致性
  - 25宫格连贯分镜

## 二、行业文本清洗最佳实践

### 2.1 预处理阶段标准流程

```
原始小说文本
    ↓
【阶段1: 格式标准化】
- 编码统一 (UTF-8)
- 换行符统一 (\n)
- BOM头移除
    ↓
【阶段2: 噪声清洗】
- 控制字符移除 (\x00-\x1F, \x7F-\x9F)
- 特殊分隔符处理 (===, ---, * * *, ///)
- ASCII艺术/装饰线移除
- 广告/推广信息过滤
    ↓
【阶段3: 内容结构化】
- 章节识别与分割
- 段落重组（对话/叙述分离）
- 元数据提取（标题、作者、简介）
    ↓
【阶段4: 语义优化】
- 停用词标记（保留但标注）
- 敏感词检测
- 重复内容检测
    ↓
清洗后文本 → AI解析引擎
```

### 2.2 特殊字符处理清单

| 字符类型   | 示例                        | 处理方式           | 优先级 |
| ---------- | --------------------------- | ------------------ | ------ |
| 章节分隔符 | `第X章`, `Chapter X`, `===` | 识别并标记章节边界 | 高     |
| 场景分隔符 | `---`, `* * *`, `///`       | 移除或替换为换行   | 高     |
| 装饰线     | `━━━━━━━━`, `────────`      | 移除               | 中     |
| 作者注     | `(作者:)`, `【注】`         | 分离为元数据       | 中     |
| 广告文本   | `请关注公众号`, `加群`      | 正则过滤移除       | 高     |
| 乱码字符   | �, □, �                     | 检测并移除         | 高     |
| 多余空格   | 连续3+空格                  | 压缩为1个          | 低     |
| HTML标签   | `<br>`, `&nbsp;`            | 实体解码/移除      | 中     |

### 2.3 章节识别算法

**多模式匹配策略**：

```regex
# 中文章节
第[一二三四五六七八九十百千零\d]+章.*
第[\d]+章.*
章节[\d]+.*

# 英文章节
Chapter\s+[\dIVXLC]+.*
CHAPTER\s+[\d]+.*

# 特殊标记
\*{3,}.*\*{3,}  # *** 章节名 ***
={3,}.*={3,}    # === 章节名 ===
```

## 三、AI解析阶段行业实践

### 3.1 分阶段解析Pipeline

#### **阶段1: 元数据提取**

```
输入: 清洗后文本
输出:
- 小说标题
- 作者
- 总字数
- 预估章节数
- 主要题材标签
```

#### **阶段2: 角色识别**

```
技术方案:
1. NER命名实体识别（人名提取）
2. 共指消解（"他"/"她"→具体角色）
3. 角色属性提取:
   - 外貌描述
   - 性格特征
   - 角色关系
   - 出场频率统计

输出:
- 角色列表（主角、配角、龙套）
- 角色关系图谱
- 角色出场时间线
```

#### **阶段3: 场景分析**

```
技术方案:
1. 场景边界检测（时间/地点变化）
2. 场景类型分类:
   - 室内/室外
   - 白天/夜晚/黄昏
   - 现代/古代/科幻
3. 场景描述提取
4. 场景情绪标注

输出:
- 场景列表
- 场景转换图
- 场景-角色关联矩阵
```

#### **阶段4: 分镜生成**

```
技术方案:
1. 动作指令识别（"站起身"、"推开门"）
2. 镜头语言检测:
   - 特写(Close-up)
   - 中景(Medium shot)
   - 全景(Wide shot)
3. 对话-动作配对
4. 情绪节奏分析

输出:
- 分镜脚本
- 镜头列表
- 拍摄建议
```

### 3.2 语义分块策略对比

| 策略         | 适用场景     | 优点         | 缺点           |
| ------------ | ------------ | ------------ | -------------- |
| **固定字数** | 通用场景     | 简单可控     | 可能切断语义   |
| **章节边界** | 结构化小说   | 保持完整性   | 章节长度差异大 |
| **场景边界** | 场景变化频繁 | 逻辑清晰     | 需要预处理识别 |
| **语义聚类** | 复杂叙事     | 最优语义连贯 | 计算成本高     |

**行业推荐**：混合策略

- 优先按章节分割
- 超长章节(>5000字)按场景二次分割
- 保证每块3000-5000字（LLM最优输入长度）

## 四、角色一致性保障方案

### 4.1 行业通用做法

#### **方案A: 角色档案系统**

```
角色基础设定提取 → 生成角色卡 → 视觉一致性约束

角色卡内容:
- 基础信息（姓名、性别、年龄）
- 外貌特征（发色、瞳色、身高、体型）
- 服装特征（主要服装、配饰）
- 性格标签
- 标志性动作
```

#### **方案B: 视觉锚点机制**

```
首镜角色图 → 作为参考图 → 后续生成使用图生图

技术实现:
1. 生成角色三视图（正/侧/背）
2. 存储角色视觉特征向量
3. 生成时注入参考图+特征向量
```

#### **方案C: 记忆银行 (MemFlow)**

```
港大MemFlow技术:
- 为AI配备智能"记忆银行"
- 存储已生成角色/场景特征
- 新提示输入时检索相似特征
- 防止AI"忘记"之前内容
```

### 5.4 智能分块优化（中优先级）

#### **当前问题**

- 固定字数分块可能切断语义
- 未考虑章节边界

#### **优化方案**

```typescript
// 混合分块策略
function smartChunk(text: string, maxChunkSize: number = 4000): string[] {
  const chapters = extractChapters(text);
  const chunks: string[] = [];

  for (const chapter of chapters) {
    if (chapter.content.length <= maxChunkSize) {
      chunks.push(chapter.content);
    } else {
      // 超长章节按场景分割
      const scenes = extractScenes(chapter.content);
      let currentChunk = '';

      for (const scene of scenes) {
        if (currentChunk.length + scene.length > maxChunkSize) {
          if (currentChunk) chunks.push(currentChunk);
          currentChunk = scene;
        } else {
          currentChunk += '\n' + scene;
        }
      }

      if (currentChunk) chunks.push(currentChunk);
    }
  }

  return chunks;
}
```
