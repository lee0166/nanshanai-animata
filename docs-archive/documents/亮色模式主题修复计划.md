# 亮色模式主题修复计划

## 问题分析

### 当前状况

1. **tailwind.config.js** 中只配置了 `dark` 主题，没有配置 `light` 主题
2. 当用户切换到亮色模式时，HeroUI 使用默认的亮色主题色（蓝色）
3. 橙红色主题只在 dark 模式下生效

### 问题根源

```javascript
// tailwind.config.js 当前配置
plugins: [heroui({
  themes: {
    dark: {
      colors: {
        primary: {
          DEFAULT: "#ea580c", // 橙红色只在 dark 主题中定义
          // ...
        }
      }
    }
    // 缺少 light 主题配置！
  }
})],
```

## 修复方案

### 方案：添加亮色主题配置

在 `tailwind.config.js` 中添加 `light` 主题配置，使用相同的橙红色主色调。

### 实施步骤

#### 步骤1: 修改 tailwind.config.js

```javascript
plugins: [heroui({
  themes: {
    light: {
      colors: {
        background: "#ffffff",
        foreground: "#1f2937",
        primary: {
          50: '#fff7ed',
          100: '#ffedd5',
          200: '#fed7aa',
          300: '#fdba74',
          400: '#fb923c',
          500: '#f97316',
          600: '#ea580c',
          700: '#c2410c',
          800: '#9a3412',
          900: '#7c2d12',
          DEFAULT: "#ea580c", // 与 dark 模式相同的橙红色
          foreground: "#FFFFFF",
        },
        secondary: {
          DEFAULT: "#71717a",
          foreground: "#FFFFFF",
        },
        content1: "#f9fafb", // 亮色模式卡片背景
        content2: "#f3f4f6",
        content3: "#e5e7eb",
        content4: "#d1d5db",
        focus: "#f97316",
      }
    },
    dark: {
      colors: {
        // ... 保持现有配置不变
      }
    }
  }
})],
```

#### 步骤2: 验证修改

1. 启动开发服务器
2. 切换到亮色模式
3. 检查按钮、图标、标签是否显示为橙红色
4. 检查背景色是否正确（白色背景）

### 工作量评估

| 任务                    | 预计时间       |
| ----------------------- | -------------- |
| 修改 tailwind.config.js | 5 分钟         |
| 验证亮色模式效果        | 5 分钟         |
| **总计**                | **约 10 分钟** |

### 风险分析

- **风险等级**: 极低
- **影响范围**: 仅影响亮色模式下的颜色显示
- **回滚方式**: 恢复 tailwind.config.js 备份

### 预期效果

- ✅ 亮色模式下按钮、标签、图标显示为橙红色
- ✅ 亮色模式下背景为白色/浅灰色
- ✅ 暗色模式保持现有效果不变
- ✅ 主题切换时颜色过渡平滑
