## 3. 完整业务流程

### 3.1 流程图概览

```
小说上传
   ↓
[1] 文本预处理与清洗
   ↓
[2] 创建剧本对象 (Script)
   ↓
[3] 解析策略选择 (Strategy Selection)
   ├─→ 短文本 (Fast Path, &lt;800字)
   ├─→ 标准文本 (Standard Path, 800-5000字)
   └─→ 长文本 (Chunked Path, &gt;5000字)
         ↓
[4] 元数据提取 (Metadata)
   ├─→ 标题、字数、预估时长
   ├─→ 角色列表、场景列表
   ├─→ 体裁、基调
   └─→ 全局上下文提取 (Global Context)
         ├─→ 故事核心 (梗概、冲突、主题)
         ├─→ 故事结构 (三幕式/英雄之旅)
         ├─→ 视觉风格 (美术指导、色彩方案)
         ├─→ 时代背景 (年代、地点)
         ├─→ 情绪曲线 (Emotional Arc)
         └─→ 一致性规则
               ↓
[5] 角色解析 (Characters, 并行处理)
   ├─→ 发现角色名称
   ├─→ 批量提取角色详情 (Batch Extraction)
   ├─→ 外貌特征 (身高、体型、面容、发型、服饰)
   ├─→ 性格特征
   ├─→ 标志性物品
   ├─→ 情绪弧线
   └─→ 人物关系
         ↓
[6] 场景解析 (Scenes, 并行处理)
   ├─→ 发现场景名称
   ├─→ 批量提取场景详情 (Batch Extraction)
   ├─→ 环境描述 (纯环境，不含人物动作)
   ├─→ 时间/季节/天气
   ├─→ 建筑风格/陈设/光线/色调
   ├─→ 场景功能
   └─→ 涉及角色
         ↓
[7] 分镜生成 (Shots)
   ├─→ 批量生成分镜 (每场景3个)
   ├─→ 景别选择 (extreme_long/long/full/medium/close_up/extreme_close_up)
   ├─→ 运镜方式 (static/push/pull/pan/tilt/track/crane)
   ├─→ 画面描述 (30字以内)
   ├─→ 台词/音效
   ├─→ 预估时长
   └─→ 涉及角色
         ↓
[8] 质量评估 (Quality Report)
   ├─→ 完整性检查
   ├─→ 一致性检查
   ├─→ 质量评分 (0-100)
   └─→ 优化建议
         ↓
[9] 数据持久化
   ├─→ 保存剧本对象
   ├─→ 保存解析状态
   └─→ 保存质量报告
         ↓
    解析完成
```

## 4. 各环节详细说明

### 4.1 环节1：小说上传与文本预处理

#### 4.1.1 小说项目信息

**文件格式**：纯文本文件 (.txt, 可选其他文本格式)
**内容特征**：
- 支持章节化小说 (第X章/集/回/节)
- 支持无章节标记的纯文本
- 支持中英文混排
- 编码：UTF-8 (自动检测编码问题)

**元数据**：
- 文件名 → 剧本标题
- 文件大小 → 用于估计处理时间
- 上传时间 → createdAt 时间戳

#### 4.1.2 文本清洗流程 (TextCleaner)

**文件位置**：`services/textCleaner.ts:29-59`

**清洗步骤**：

| 步骤 | 操作 | 代码位置 |
|------|------|---------|
| 1 | 统一换行符 (\\r\\n→\\n, \\r→\\n) | `textCleaner.ts:37-38` |
| 2 | 去除控制字符 (保留\\n, \\t) | `textCleaner.ts:41` |
| 3 | 合并多余空行 (3+→2) | `textCleaner.ts:44` |
| 4 | 去除行首行尾空白 | `textCleaner.ts:47` |
| 5 | 统一空格 (多个→1个) | `textCleaner.ts:50` |
| 6 | 去除空行中的空格 | `textCleaner.ts:53` |
| 7 | 去除全文首尾空白 | `textCleaner.ts:56` |

**章节提取规则** (`textCleaner.ts:66-122`)：

支持的章节标题模式：
- `第X章/集/回/节 + 标题`
- `Chapter X + 标题`
- `X. + 标题`

**产出物**：
```typescript
interface TextCleanResult {
  cleanedText: string;           // 清洗后的文本
  chapters: Chapter[];           // 提取的章节列表
  stats: {
    originalLength: number;      // 原始字符数
    cleanedLength: number;       // 清洗后字符数
    chapterCount: number;        // 章节数量
    removedChars: number;        // 移除的字符数
  };
}
```

**触发条件**：用户在ScriptManager页面点击上传按钮 → `handleFileUpload()` (`ScriptManager.tsx:260-298`)

### 4.3 环节3：解析策略选择

**文件位置**：`services/scriptParser.ts:3256-3277`

**策略选择器** (`ParseStrategySelector`)：

| 策略 | 触发条件 | 预估时间 | 推荐批大小 |
|------|---------|---------|-----------|
| **Fast Path** | 字数 &lt; 800 | 60s | 10 |
| **Standard Path** | 800-5000字 | 动态计算 | 5 |
| **Chunked Path** | 字数 &gt; 5000 | 360s+ | 3 |

**字数计算**：`content.length` (中文字符=1字)

**选择逻辑代码** (`scriptParser.ts:3258-3276`)：
```typescript
const strategySelection = this.strategySelector.selectStrategy(content);
// strategySelection.strategy: 'fast' | 'standard' | 'chunked'
```

**Chunked Path分块规则**：
- 最大块大小：6000字符 (`CONFIG.maxChunkSize`)
- 语义分块：保留章节完整性

### 4.4 环节4：元数据提取与全局上下文

#### 4.4.1 元数据提取 (Metadata)

**文件位置**：`services/scriptParser.ts:1745-1777`

**输入数据**：完整清洗文本 `content`

**模型调用参数**：
- Model: doubao-seed-1-6-251015 (配置的LLM模型)
- Max Tokens: 4000
- Temperature: 0.3
- JSON Mode: true

**Prompt** (`scriptParser.ts:334-371`)：
```
请全面分析以下剧本/小说内容，提取完整的元数据：

【剧本内容】
{content}

请提取：
1. 作品标题
2. 总字数（估算）
3. 预估时长（如"10分钟"）
4. 角色总数量
5. 所有角色名称列表
6. 场景总数量
7. 所有场景名称列表
8. 章节/幕数
9. 故事类型
10. 整体基调
```

**产出物** (`ScriptMetadata`, `types.ts:426-489`)：

### 4.5 环节5：角色解析 (Characters)

**文件位置**：`services/scriptParser.ts:1858-2300`

**触发条件**：元数据提取完成，`stage === 'characters'`

**处理流程**：

```

   ↓
过滤已处理的角色
   ↓
批量提取角色详情 (Batch Extraction)

   ↓
LLM调用 → 解析角色
   ↓
生成视觉Prompt
   ↓
保存到parseState.characters
```

**批处理逻辑** (`scriptParser.ts:2254-2280`)：
- 所有剩余角色一次性批量提取

**Prompt模板** (`scriptParser.ts:416-459`)：

### 4.6 环节6：场景解析 (Scenes)

**文件位置**：`services/scriptParser.ts:1905-2463`

**触发条件**：角色解析完成，与角色解析**并行执行**

**处理流程与角色解析类似**，关键区别在于：

#### 4.6.1 场景描述严格规则

**文件位置**：`scriptParser.ts:504-523`

**禁止内容**（必须严格遵守）：
- ❌ 人物名称
- ❌ 人物动作 (坐、站、走、跑、拿、握等)
- ❌ 剧情描述 (对峙、谈判、表白、冲突等)
- ❌ 情感/氛围形容词 (紧张、浪漫、悲伤等)
- ❌ 时间状语从句 (当...时、在...过程中)
- ❌ 指代词 (在此、这里、此处)

**正确示例**：
```
"现代商务办公室，配备实木办公桌、皮质转椅和落地书架"
"中型会议室，长方形会议桌配12把黑色皮椅，墙面挂有企业标识"
```

**错误示例**：
```
"沈若涵坐在办公桌后处理文件" (含人名和动作)
"宽敞明亮的会议室，见证了职场博弈的最终落幕" (含剧情)
```

#### 4.7.2 分镜数据结构

**文件位置**：`types.ts:542-557`

```typescript
interface Shot {
  id: string;
  sequence: number;              // 序号
  sceneName: string;            // 所属场景
  sceneId?: string;
```
