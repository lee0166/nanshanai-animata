# 智能分块优化与章节识别增强方案

## 一、项目背景与目标

### 当前问题

1. **分块策略简单**：仅按字符数（6000字符）和段落边界分块，导致语义截断
2. **章节识别有限**：仅支持标准格式（第X章、Chapter X），对网络小说格式支持不足
3. **用户体验影响**：长文本解析时角色/场景提取不完整，需要手动调整

### 优化目标

- 实现语义感知的智能分块，保持场景/对话完整性
- 支持多种章节格式，自动识别并提取章节结构
- 提升解析质量和用户体验

#### 2.1.2 分块优先级（从高到低）

1. **章节边界** - 检测到章节标题时强制分块
2. **场景转换** - 检测到场景切换标记（时间、地点变化）
3. **对话结束** - 对话段落结束后
4. **段落边界** - 自然段落结束
5. **强制截断** - 达到maxChunkSize时强制分块

### 2.2 章节识别增强

#### 2.2.1 支持的章节格式

```typescript
const chapterPatterns = [
  // 标准中文格式
  { pattern: /^第[一二三四五六七八九十百千万\d]+章[\s:：]*/, weight: 1.0 },
  { pattern: /^第\d+章[\s:：]*/, weight: 1.0 },
  { pattern: /^Chapter\s+\d+[\s:：]*/i, weight: 1.0 },
  { pattern: /^CHAPTER\s+\d+[\s:：]*/, weight: 1.0 },

  // 卷/集格式
  { pattern: /^第[一二三四五六七八九十百千万\d]+卷[\s:：]*/, weight: 0.9 },
  { pattern: /^第[一二三四五六七八九十百千万\d]+集[\s:：]*/, weight: 0.9 },
  { pattern: /^第[一二三四五六七八九十百千万\d]+回[\s:：]*/, weight: 0.9 },
  { pattern: /^第[一二三四五六七八九十百千万\d]+节[\s:：]*/, weight: 0.8 },

  // 数字格式
  { pattern: /^\d+[\.、\s]+[^\d]/, weight: 0.7 }, // 1. 或 1、开头
  { pattern: /^\d+\s*$/, weight: 0.5 }, // 纯数字行

  // 特殊符号包裹
  { pattern: /^[【\[\(（]第?[一二三四五六七八九十百千万\d]+章?[】\]\)）]/, weight: 0.8 },
  { pattern: /^#+\s*第?[一二三四五六七八九十百千万\d]+章?/, weight: 0.8 }, // Markdown

  // 无数字标题（需要上下文判断）
  { pattern: /^[前言|序章|引言|楔子|尾声|后记|番外]/, weight: 0.6 },
];
```

#### 2.2.2 章节信息提取

```typescript
interface ChapterInfo {
  index: number; // 章节序号（从1开始）
  title: string; // 章节标题
  level: number; // 层级（1=章，2=节）
  startPos: number; // 在原文中的起始位置
  endPos: number; // 在原文中的结束位置
  wordCount: number; // 字数
  format: string; // 检测到的格式类型
  confidence: number; // 置信度（0-1）
}
```

#### 2.2.3 智能章节检测算法

```typescript
function detectChapters(text: string): ChapterInfo[] {
  const lines = text.split('\n');
  const chapters: ChapterInfo[] = [];
  let currentChapter: Partial<ChapterInfo> | null = null;

  for (let i = 0; i < lines.length; i++) {
    const line = lines[i].trim();

    // 尝试匹配所有章节模式
    for (const { pattern, weight } of chapterPatterns) {
      if (pattern.test(line)) {
        // 如果已有章节，先结束它
        if (currentChapter) {
          currentChapter.endPos = lines.slice(0, i).join('\n').length;
          chapters.push(currentChapter as ChapterInfo);
        }

        // 开始新章节
        currentChapter = {
          index: chapters.length + 1,
          title: extractChapterTitle(line),
          startPos: lines.slice(0, i).join('\n').length,
          format: pattern.source,
          confidence: weight,
        };
        break;
      }
    }
  }

  // 结束最后一个章节
  if (currentChapter) {
    currentChapter.endPos = text.length;
    chapters.push(currentChapter as ChapterInfo);
  }

  return validateChapters(chapters);
}
```

## 六、风险评估与应对

### 风险1：章节格式过于多样导致误识别

**应对**：

- 引入置信度机制，低置信度时提示用户确认
- 支持用户手动标记章节格式
- 提供章节预览功能供用户检查

### 风险2：智能分块导致分块过大/过小

**应对**：

- 设置最小/最大分块大小限制
- 添加分块质量评估，质量低时给出警告
- 允许用户调整分块策略参数

### 风险3：性能问题（大文本处理慢）

**应对**：

- 使用流式处理大文本
- 添加进度提示
- 优化正则匹配性能
