# 分镜数据删除行为分析

## 问题描述

用户反馈：删除了剧本后，分镜管理页面看不到对应剧本的分镜内容。

## 分析结果

### 这是预期的正确行为，不是 Bug

### 1. 分镜数据的存储方式

**分镜数据是存储在剧本内部的**，不是独立存储。

```typescript
// types.ts 中 Script 和 ScriptParseState 的定义
export interface Script {
  id: string;
  projectId: string;
  title: string;
  content: string;
  parseState: ScriptParseState; // 分镜数据在这里
  createdAt: number;
  updatedAt: number;
}

export interface ScriptParseState {
  stage: ParseStage;
  progress: number;
  characters?: ScriptCharacter[];
  scenes?: ScriptScene[];
  items?: ScriptItem[];
  shots?: Shot[]; // ← 分镜数据作为剧本的一部分
  // ...
}
```

### 2. 数据关系图

```
Project
  └── scripts_{projectId}.json
        └── Script[]
              └── Script.parseState
                    ├── characters: ScriptCharacter[]
                    ├── scenes: ScriptScene[]
                    ├── items: ScriptItem[]
                    └── shots: Shot[]  ← 分镜数据内嵌在剧本中

  └── assets_{projectId}.json
        └── Asset[] (通过 scriptId 外键关联到剧本)
```

### 3. 为什么分镜"消失"了？

| 数据类型           | 存储位置                   | 关联方式                         | 删除剧本时的行为       |
| ------------------ | -------------------------- | -------------------------------- | ---------------------- |
| **角色/场景/物品** | `assets_{projectId}.json`  | 通过 `scriptId` 外键关联         | 需要级联删除（已实现） |
| **分镜**           | `scripts_{projectId}.json` | 内嵌在 `Script.parseState.shots` | **随剧本一起删除**     |

**关键点**：

- 角色、场景、物品是**独立存储**的资产，通过 `scriptId` 关联到剧本
- 分镜是**内嵌存储**在剧本对象中的，是剧本的一部分
- 删除剧本时，整个剧本对象（包括 `parseState` 及其中的 `shots`）都被删除了

### 4. 分镜管理页面的数据加载逻辑

```typescript
// ShotManager.tsx
const currentScript = useMemo(() => {
  return scripts.find(s => s.id === selectedScriptId);
}, [scripts, selectedScriptId]);

const allShots = useMemo(() => {
  return currentScript?.parseState?.shots || [];
}, [currentScript]);
```

分镜管理页面：

1. 先加载剧本列表 `storageService.getScripts(projectId)`
2. 根据选中的剧本 ID 找到当前剧本
3. 从 `currentScript.parseState.shots` 读取分镜数据

**当剧本被删除后**：

- `getScripts()` 返回的列表中不再包含该剧本
- 分镜管理页面自然无法获取到对应的分镜数据
- 这不是"过滤"，而是数据确实不存在了

## 结论

### 当前行为是正确的

1. **分镜数据确实被删除了** - 因为它存储在剧本对象内部，剧本删除时分镜也随之删除
2. **前端看不到分镜是预期行为** - 不是过滤，而是数据已不存在
3. **不需要额外的级联删除逻辑** - 分镜不需要像角色/场景/物品那样单独删除

### 是否需要优化？

**建议：在删除确认对话框中增加分镜数量提示**

虽然分镜会随剧本自动删除，但为了用户知情权，建议在删除确认对话框中显示：

- 角色数量
- 场景数量
- 物品数量
- **分镜数量**（新增）

这样用户可以清楚知道删除剧本会影响哪些数据。

## 优化建议

### 方案：在删除确认对话框中显示分镜数量

修改 `ScriptManager.tsx` 中的 `openDeleteModal` 方法：

```typescript
const openDeleteModal = async (script: Script) => {
  setScriptToDelete(script);
  setIsDeleteModalOpen(true);

  // 获取关联资源统计
  try {
    const assets = await storageService.getAssets(projectId!);
    const relatedAssets = assets.filter(a => a.scriptId === script.id);

    // 计算分镜数量（从剧本的 parseState 中读取）
    const shotCount = script.parseState?.shots?.length || 0;

    setDeleteStats({
      characters: relatedAssets.filter(a => a.type === AssetType.CHARACTER).length,
      scenes: relatedAssets.filter(a => a.type === AssetType.SCENE).length,
      items: relatedAssets.filter(a => a.type === AssetType.ITEM).length,
      shots: shotCount, // 新增分镜数量
    });
  } catch (error) {
    console.error('Failed to get delete stats:', error);
    setDeleteStats({ characters: 0, scenes: 0, items: 0, shots: 0 });
  }
};
```

这样用户删除剧本时，会清楚看到：

- "此操作将同时删除：X 个角色、Y 个场景、Z 个物品、W 个分镜"
