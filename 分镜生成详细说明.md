## 一、概述

### 1.1 什么是分镜生成

分镜生成是AI影视资产生成平台的核心功能之一，它将小说/剧本文本通过大语言模型（LLM）解析，自动提取文学要素并转换为影视级分镜脚本。每个分镜包含景别、运镜、画面描述、时长等影视制作所需的关键信息。

### 1.2 系统架构概览

## 二、分镜生成的核心规则

### 2.1 文学要素提取规则

#### 2.1.1 元数据提取（17个字段）

| 层级       | 字段              | 说明               | 示例                   |
| ---------- | ----------------- | ------------------ | ---------------------- |
| 基础信息   | title             | 作品标题           | "宫墙柳"               |
|            | wordCount         | 总字数             | 15000                  |
|            | estimatedDuration | 预估时长           | "10分钟"               |
|            | characterCount    | 角色数量           | 8                      |
|            | characterNames    | 角色名称列表       | ["沈若涵", "顾衍之"]   |
|            | sceneCount        | 场景数量           | 12                     |
|            | sceneNames        | 场景名称列表       | ["御书房", "冷宫"]     |
|            | chapterCount      | 章节数             | 5                      |
|            | genre             | 类型/题材          | "古装宅斗"             |
|            | tone              | 整体基调           | "爽剧"                 |
| 故事核心   | synopsis          | 故事梗概           | 100-200字              |
|            | logline           | 一句话简介         | 30字以内               |
|            | coreConflict      | 核心冲突           | 主要矛盾描述           |
|            | theme             | 主题标签           | ["爱情", "复仇"]       |
| 故事结构   | storyStructure    | 三幕式/英雄之旅    | 包含act1-3、中点、高潮 |
| 视觉风格   | visualStyle       | 美术/摄影/光影风格 | 色彩调色板、情绪       |
| 时代背景   | eraContext        | 年代、地点、季节   | "唐代长安"             |
| 情绪曲线   | emotionalArc      | 情绪变化节点       | 情节点+情绪+强度+色调  |
| 一致性规则 | consistencyRules  | 角色特征、时代限制 | 用于后续校验           |
| 参考信息   | references        | 参考影片/导演      | ["《甄嬛传》"]         |

#### 2.1.2 角色提取规则

```typescript
interface ScriptCharacter {
  name: string; // 角色名
  gender: 'male' | 'female' | 'unknown';
  age: string; // 年龄（纯数字）
  identity: string; // 身份/职业
  appearance: {
    height: string; // 身高描述
    build: string; // 体型
    face: string; // 面容特征
    hair: string; // 发型
    clothing: string; // 服饰（含脚部推断）
  };
  personality: string[]; // 性格特征
  signatureItems: string[]; // 标志性物品（过滤临时物品）
  emotionalArc: Array<{ phase: string; emotion: string }>;
  relationships: Array<{ character: string; relation: string }>;
  visualPrompt: string; // 生图提示词（自动生成）
}
```

**标志性物品过滤规则**（不包含临时物品）：

- ❌ 排除：文件、笔记本、咖啡、手机、纸张、杯子、笔、书本、包、食物、饮料
- ✅ 保留：玉佩、长剑、折扇、眼镜、戒指、项链、武器

#### 2.1.3 场景提取规则

**场景描述严格约束**：

```
【禁止包含】
1. 人物名称
2. 人物动作（坐、站、走、跑、拿、握等）
3. 剧情描述（对峙、谈判、表白、冲突等）
4. 情感形容词（紧张、浪漫、悲伤、激烈等）
5. 时间状语从句（当...时、在...过程中等）
6. 指代词（在此、这里、此处）

【正确示例】
✓ "现代商务办公室，配备实木办公桌、皮质转椅和落地书架"
✓ "中型会议室，长方形会议桌配12把黑色皮椅，墙面挂有企业标识"

【错误示例】
✗ "沈若涵坐在办公桌后处理文件"（包含人名和动作）
✗ "紧张压抑的审讯室，充满对抗氛围"（包含情感形容词）
```

### 2.2 视觉化转换标准

#### 2.2.1 景别标准（ShotType）

| 景别   | 英文标识         | 说明              | 适用场景           |
| ------ | ---------------- | ----------------- | ------------------ |
| 极远景 | extreme_long     | 展示宏大环境      | 战争场面、自然风光 |
| 远景   | long             | 人物占画面1/4以下 | 环境交代、氛围营造 |
| 全景   | full             | 人物全身          | 动作展示、舞蹈     |
| 中景   | medium           | 人物膝盖以上      | 对话场景、日常互动 |
| 近景   | close_up         | 人物胸部以上      | 表情刻画、情感交流 |
| 特写   | extreme_close_up | 面部或局部        | 细节强调、情绪高潮 |

| 静止   | static    | 固定机位         |
| 推     | push      | 镜头前移靠近主体 |
| 拉     | pull      | 镜头后移远离主体 |

| 跟     | track     | 跟随主体移动     |
| crane  | crane     | 大幅度升降       |
| 变焦推 | zoom_in   | 焦距变化靠近     |
| 变焦拉 | zoom_out  | 焦距变化远离     |

| 角度 | 英文标识    | 视觉效果   |
| ---- | ----------- | ---------- |
| 平视 | eye_level   | 自然、平等 |

### 2.3 场景划分原则

#### 2.3.1 场景识别策略

系统使用三级定位策略从全文中提取场景上下文：

1. **名称精确匹配**（优先级最高）
   - 匹配模式：`场景名`、`【场景名】`、`场景：场景名`、`Scene: 场景名`

2. **描述关键词匹配**
   - 提取场景描述中的关键词（过滤停用词）
   - 需要至少匹配2个关键词才确认

3. **模糊匹配**（环境信息）
   - 使用建筑风格、陈设、时间段、天气等辅助定位

#### 2.3.2 上下文提取范围

```typescript
interface SceneContextConfig {
  defaultContextChars: 500; // 默认前后各500字符
  minContextChars: 100; // 最小100字符
  maxContextChars: 1000; // 最大1000字符
}
```

**智能截断规则**：

- 在段落边界处截断（`\n\n`、句号、感叹号等）
- 保持句子完整性
- 最多前后查找100字符寻找边界

### 2.4 分镜类型定义规范

#### 2.4.1 分镜内容类型（ShotContentType）

| 类型     | 标识            | 关键帧数 | 适用场景         | 生成策略                     |
| -------- | --------------- | -------- | ---------------- | ---------------------------- |
| 静态     | static          | 1        | 对话、环境展示   | 生成图片                     |
| 简单动态 | dynamic-simple  | 2        | 走路、转身、开门 | 生成视频（首帧+尾帧）        |
| 复杂动态 | dynamic-complex | 3        | 打斗、追逐、特效 | 生成视频（首帧+中间帧+尾帧） |

#### 2.4.2 自动类型检测规则

**复杂动态关键词**（需要3个关键帧）：

```
打斗、战斗、追逐、奔跑、跳跃、翻滚、爆炸、特效
fight、battle、chase、run、jump、roll、explosion
effect、激烈、快速移动、连续动作、复杂的、激烈地
```

**简单动态关键词**（需要2个关键帧）：

```
走、走动、转身、回头、坐下、站起、开门、关门
walk、turn、sit、stand、open、close、move、enter
exit、走向、离开、靠近、拿起、放下
```

**运镜判断**：

- 动态运镜（推/拉/摇/移/跟）→ 简单动态
- 静止运镜 → 静态

#### 2.4.3 分镜层级（ShotLayer）

| 层级     | 标识     | 说明                               |
| -------- | -------- | ---------------------------------- |
| 关键分镜 | key      | 必须生成，对应情节点、转折点、高潮 |
| 可选分镜 | optional | 按需生成，用于过渡、环境展示       |

### 3.3 关键决策节点

```
决策节点1: 解析策略选择
├─ 短剧本 (< 3000字) ──▶ Fast Path (并行提取)
├─ 中剧本 (3000-8000字) ──▶ Standard Path
└─ 长剧本 (> 8000字) ──▶ Deep Path (语义分块)

决策节点2: 分镜数量计算
├─ 场景复杂度分析 (0-10分)
│   ├─ 场景长度 (30%)
│   ├─ 角色数量 (25%)
│   ├─ 动作元素 (25%)
│   └─ 环境复杂度 (20%)
├─ 基础分镜数: 8个
├─ 复杂度加成: 每1分 +1.5个
└─ 范围限制: 8-30个

决策节点3: 关键帧数量确定
├─ 静态分镜 ──▶ 1个关键帧
├─ 简单动态 ──▶ 2个关键帧 (start + end)
└─ 复杂动态 ──▶ 3个关键帧 (start + middle + end)

决策节点4: 图片生成模式
├─ 文生图 ──▶ 仅使用提示词
└─ 参考图生图 ──▶ 提示词 + 角色参考图 + 场景参考图
```

## 四、完整流程说明

### 4.1 阶段一：小说上传与文本预处理

#### 4.1.1 文件上传处理

**入口**: `views/ScriptManager.tsx`

```typescript
// 支持的文件格式
const SUPPORTED_FORMATS = ['.txt', '.md', '.docx', '.pdf'];

// 文件处理流程
1. 文件选择 ──▶ 2. 格式验证 ──▶ 3. 文本提取 ──▶ 4. 存储
```

**文本提取逻辑**:

- `.txt/.md`: 直接读取文本内容
- `.docx`: 使用docx解析库提取段落
- `.pdf`: 使用pdf解析库提取文本

#### 4.1.2 语义分块（SemanticChunking）

**文件**: `services/parsing/SemanticChunker.ts`

```typescript
interface SemanticChunk {
  id: string;
  content: string; // 文本内容
  startIndex: number; // 起始位置
  endIndex: number; // 结束位置
  type: 'scene' | 'dialogue' | 'action' | 'description';
  characters: string[]; // 涉及角色
  sceneContext?: string; // 场景上下文
}
```

**分块策略**:

- 按段落分隔（`\n\n`）
- 按场景标记（"场景X"、"第X章"）
- 最大块大小: 6000字符
- 保留段落完整性

#### 4.2.5 分镜生成（Shot Generation）

**智能分镜生成器**: `services/parsing/IntelligentShotGenerator.ts`

**复杂度分析算法**:

```typescript
function analyzeComplexity(scene, characters): SceneComplexity {
  const factors = [
    { name: '场景长度', weight: 0.3, score: scoreByLength(description.length) },
    { name: '角色数量', weight: 0.25, score: scoreByCharacters(characters.length) },
    { name: '动作元素', weight: 0.25, score: scoreByAction(description) },
    { name: '环境复杂度', weight: 0.2, score: scoreByEnvironment(description) },
  ];

  const totalScore = factors.reduce((sum, f) => sum + f.score * f.weight, 0);
  return { score: Math.round(totalScore), factors, level: getLevel(totalScore) };
}
```

**分镜数量计算**:

```
基础分镜数: 8个
复杂度加成: complexityScore × 1.5
计算公式: optimalShots = floor(max(8, min(30, 8 + complexityScore × 1.5)))
```

**Token预算计算**:

```
基础开销: 2000 tokens
每分镜: 180 tokens
安全余量: 1.2倍
计算公式: requiredTokens = ceil((2000 + shots × 180) × 1.2)
范围限制: 3000-9000 tokens
```
